# 实时监听与缓存架构文档

## 一、整体架构概述

本小程序采用**分层缓存 + 实时监听**的架构，实现商品数据的实时更新与高性能展示。

## 二、全局监听器

### 2.1 位置与启动

**文件**: `miniprogram/app.js`

**启动时机**: 登录成功后启动

```javascript
// app.js
onLaunch() {
  // 登录成功后启动全局商品监听器
  this.onLoginReady(() => {
    this.startGlobalProductWatcher();
  });
}
```

### 2.2 监听范围

```javascript
// 监听所有未删除的商品
db.collection('products')
  .where({ isDeleted: false })
  .watch({ ... })
```

### 2.3 监听器回调处理

```javascript
onChange: (snapshot) => {
  // 引入缓存工具
  const cache = require('./utils/cache');

  if (snapshot.type === 'init') {
    // =============================================
    // 场景：初始化快照 / 重连时收到的快照
    // =============================================
    // 更新全部缓存
    if (snapshot.docs && snapshot.docs.length > 0) {
      snapshot.docs.forEach(doc => {
        cache.cacheProduct(doc._id, doc);
      });
    }
  } else {
    // =============================================
    // 场景：数据变化（管理员修改/用户支付/取消订单）
    // =============================================
    // 增量更新缓存
    if (snapshot.docChanges && snapshot.docChanges.length > 0) {
      snapshot.docChanges.forEach(change => {
        if (change.dataType === 'update' || change.dataType === 'add') {
          if (change.doc) {
            cache.cacheProduct(change.doc._id, change.doc);
          }
        }
        // remove 类型暂不处理（标记 isDeleted）
      });
    }
  }
}
```

### 2.4 错误处理与重连

```javascript
onError: (error) => {
  console.error('[全局监听器] 监听出错:', error);
  // 清理引用
  this._globalProductWatcher = null;

  // 延迟 3 秒重连
  setTimeout(() => {
    console.log('[全局监听器] 尝试重新连接...');
    this.startGlobalProductWatcher();
  }, 3000);
}
```

### 2.5 全局监听器职责总结

| 职责 | 说明 |
|------|------|
| **初始化** | 小程序启动时，收到 init 快照，更新全部缓存 |
| **实时更新** | 数据变化时（update/add），更新对应商品的缓存 |
| **重连同步** | 网络断开重连后，收到 init 快照，同步最新数据 |
| **不更新 UI** | 只负责更新缓存，不涉及页面 UI |

## 三、全局商品缓存

### 3.1 缓存结构

**文件**: `miniprogram/utils/cache.js`

```javascript
// 每个商品独立存储
tta_product_{商品ID} → {
  data: { _id, name, price, stock, status, coverImage, ... },
  expiry: 0  // 0 表示永久
}
```

### 3.2 缓存操作函数

| 函数 | 说明 | 参数 |
|------|------|------|
| `cacheProduct(id, product)` | 缓存商品 | 商品ID, 商品完整信息 |
| `getCachedProduct(id)` | 获取单个商品缓存 | 商品ID |
| `getCachedProducts(ids)` | 批量获取商品缓存 | 商品ID数组 |
| `getGlobalProductCache()` | 获取所有商品缓存 | 无（返回 Map） |

### 3.3 二级缓存策略

```javascript
// 内存缓存（热启动优化）
const productMemoryCache = new Map();

// 本地存储（持久化）
wx.setStorageSync('tta_product_xxx', { data: {...}, expiry: 0 });

// getCachedProduct 逻辑
function getCachedProduct(productId) {
  // 1. 先查内存缓存（最快）
  if (productMemoryCache.has(productId)) {
    return productMemoryCache.get(productId);
  }

  // 2. 再查本地存储
  const product = getCache(`product_${productId}`);
  if (product) {
    // 存入内存
    productMemoryCache.set(productId, product);
    return product;
  }

  // 3. 都没有，返回 null
  return null;
}
```

## 四、页面监听器

### 4.1 生命周期管理

```
┌──────────────────────────────────────────────────────────────┐
│                    页面生命周期                                │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  onLoad() ──────┐                                            │
│                  │                                            │
│  onShow() ──────┼──→ startWatchers() ──→ 创建监听器         │
│                  │                        ↓                    │
│  onHide() ──────┼──→ stopWatchers()  ←── 销毁监听器         │
│                  │                                            │
│  onUnload() ────┘        destroyWatchers()                   │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

### 4.2 监听器创建时机

**首次进入页面**:
1. `onLoad()` → `startWatchers()`
2. 等待登录成功（避免 WebSocket 未认证）

**页面返回**:
1. `onShow()` 检测 `pageVisible`
2. 如果从隐藏恢复 → 重新创建监听器

### 4.3 监听器关闭时机

| 生命周期 | 行为 | 说明 |
|----------|------|------|
| `onHide()` | `stopWatchers()` | 关闭监听器，节省资源 |
| `onUnload()` | `destroyWatchers()` | 完全销毁 |

### 4.4 监听器回调处理

```javascript
onChange: (snapshot) => {
  // =============================================
  // 场景 1：页面不可见（切换到其他页面）
  // =============================================
  if (!this.data.pageVisible) {
    this.setData({ pendingRefresh: true });  // 标记待刷新
    return;                                   // 不更新 UI
  }

  // =============================================
  // 场景 2：收到 init 快照（监听器初始化/重连）
  // =============================================
  if (snapshot.type === 'init') {
    console.log('收到 init 快照，跳过');  // 不处理
    return;
  }

  // =============================================
  // 场景 3：真实数据变化
  // =============================================
  this.handleProductChanges(snapshot);  // 增量更新 UI
}
```

#### 5.1.6 syncGlobalCacheToHomeCache 流程

```javascript
syncGlobalCacheToHomeCache(homeData, globalProductCache) {
  // 必须深拷贝
  const clonedData = JSON.parse(JSON.stringify(homeData));

  // 辅助函数：更新单个商品
  const updateProduct = (product) => {
    const updated = globalProductCache[product._id];
    if (updated) {
      return {
        ...updated,
        isOutOfStock: updated.stock <= 0 && updated.status === 'on',
        isOffline: updated.status !== 'on'
      };
    }
    return product;
  };

  // 更新 newProducts
  if (clonedData.newProducts) {
    clonedData.newProducts = clonedData.newProducts.map(updateProduct);
  }

  // 更新 seriesList 中的商品
  if (clonedData.seriesList) {
    clonedData.seriesList = clonedData.seriesList.map(series => ({
      ...series,
      products: series.products.map(updateProduct)
    }));
  }

  return clonedData;
}
```

      // 添加到 newProducts
    } else if (dataType === 'remove') {
      // 从列表移除
    }
  }

```javascript
handleProductChanges(snapshot) {
  // 跳过 init 类型
  if (dataType === 'init') {
    console.log('[宝贝页面] 跳过init类型变化');
    return;
  }

  // 增量更新 UI
  for (const change of snapshot.docChanges) {
    const { dataType, doc, docId } = change;

    if (dataType === 'update') {
      // 更新 originalProducts
      const index = originalProducts.findIndex(p => p._id === docId);
      if (index !== -1) {
        originalProducts[index] = { ...doc, ... };
      }
    } else if (dataType === 'add') {
      // 添加到 originalProducts
      originalProducts.push({ ...doc, ... });
    } else if (dataType === 'remove') {
      // 移除
      originalProducts = originalProducts.filter(p => p._id !== docId);
    }
  }

  // 更新 UI
  this.setData({ originalProducts, products: originalProducts });

  // 更新缓存
  this.setCachedProducts(originalProducts);
}
```

async fetchCartItems({ showLoading = true }) {
  // 1. 显示 loading（可选）
  if (showLoading) wx.showLoading({ title: '加载中...' });

  // 2. 获取购物车数据
  const cartRes = await db.collection('cart').where({ openid }).get();
  const cartItems = cartRes.data;

  // 3. 获取商品详情（从全局商品缓存获取）
  const productIds = cartItems.map(item => item.productId);
  const productMap = await getProductsDetail(productIds, true);

  // 4. 合并购物车数据与商品详情
  const updatedItems = cartItems.map(item => {
    const product = productMap.get(item.productId);
    return {
      ...item,
      name: product.name,
      price: product.price,
      stock: product.stock,
      coverImage: product.coverImage,
      isSoldOut: product.stock <= 0 || product.status !== 'on'
    };
  });

  // 5. 更新 UI
  this.setData({
    cartItems: updatedItems,
    filteredCartItems: updatedItems
  });

  // 6. 更新缓存
  this.setCartCache(updatedItems);
}
```

  // ===== 步骤 3：后台从数据库同步最新数据（兜底）=====
  this.fetchProduct(true);
}
```

#### 5.4.2 监听器回调

```javascript
handleProductChanges(snapshot) {
  // 跳过 init 类型
  if (snapshot.type === 'init') {
    console.log('[商品详情页面] 收到 init 快照，跳过');
    return;
  }

  // 更新 UI
  const updatedProduct = snapshot.docChanges[0].doc;
  this.setData({ product: updatedProduct });
  this.updateProductDisplay(updatedProduct);
}
```

## 六、页面间切换与数据同步

### 6.1 场景分析

#### 场景 1：用户在首页，管理员修改商品

```
时间线：
  T1: 用户在首页，页面监听器运行中
  T2: 管理员修改商品
  T3: 全局监听器捕获变化 → 更新全局商品缓存
  T4: 页面监听器捕获变化 → 增量更新 UI
  T5: 用户看到实时更新的商品信息
```

**数据流**:
```
全局监听器 → cacheProduct() → 全局商品缓存
     ↓
页面监听器 → handleProductIncrementalUpdate() → setData() → UI 更新
```

#### 场景 2：用户在首页，切换到宝贝页

```
时间线：
  T1: 首页 onHide() → stopWatchers() → 页面监听器关闭
  T2: 宝贝页 onShow() → startWatchers() → 页面监听器启动
  T3: 全局监听器持续运行
```

**数据流**:
```
全局监听器（持续运行）
     ↓
切换页面时不变
     ↓
宝贝页页面监听器启动
     ↓
监听后续变化
```

#### 场景 3：用户在首页，切换到宝贝页，管理员修改商品，用户返回首页

```
时间线：
  T1: 用户在首页
  T2: 首页 onHide() → stopWatchers()
  T3: 切换到宝贝页
  T4: 宝贝页 onShow() → startWatchers()
  T5: 管理员修改商品
  T6: 全局监听器捕获变化 → 更新全局商品缓存
  T7: 切换回首页
  T8: 首页 onShow() → loadProducts() → 同步全局缓存 → 更新 UI
```

**关键点**:
- 页面隐藏期间，全局监听器持续运行，更新全局商品缓存
- 页面返回时，`loadProducts()` 从全局商品缓存同步最新数据

**数据流**:
```
T5: 全局监听器 → cacheProduct() → 全局商品缓存（已更新）
T8: 首页 loadProducts()
     ↓
  getGlobalProductCache() → 获取最新缓存
     ↓
  syncGlobalCacheToHomeCache() → 同步到首页缓存
     ↓
  setData() → UI 显示最新数据
```

## 八、关键代码注意事项

### 8.1 深拷贝问题

**问题**: 真机环境下，`wx.getStorageSync()` 返回的对象是**只读**的，不能直接修改。

**解决方案**: 读取后立即深拷贝

```javascript
// 错误写法（真机会报错）
const cachedData = wx.getStorageSync('homeData');
cachedData.newProducts = cachedData.newProducts.map(...);  // ❌ 报错！

// 正确写法
const rawCachedData = wx.getStorageSync('homeData');
const cachedData = rawCachedData ? JSON.parse(JSON.stringify(rawCachedData)) : {};
cachedData.newProducts = cachedData.newProducts.map(...);  // ✅ 正常
```

### 8.2 监听器重复创建

**问题**: 如果 `startWatchers()` 被多次调用，可能创建多个监听器。

**解决方案**: 使用 `watcherManager` 管理监听器生命周期

```javascript
// watcherManager 会检查是否已存在，存在则不重复创建
watcherManager.create('home_products', createProductsWatcher);

// destroy 时也会清理所有引用
watcherManager.destroy('home_products');
```

### 8.3 init 快照处理

**问题**: 监听器创建时或重连时会收到 `init` 快照，包含所有数据。

**处理策略**:
- 页面首次加载：已从缓存读取，init 快照可跳过
- 页面监听器重连：UI 已是最新，init 快照可跳过

```javascript
if (snapshot.type === 'init') {
  console.log('收到 init 快照，跳过');
  return;
}
```

### 8.4 页面不可见时监听处理

**问题**: 页面隐藏时，监听器仍在运行，会触发回调。

**解决方案**: 检查 `pageVisible` 状态

```javascript
onChange: (snapshot) => {
  if (!this.data.pageVisible) {
    this.setData({ pendingRefresh: true });  // 标记待刷新
    return;  // 不更新 UI
  }
  // 正常更新 UI
}
```

## 九、缓存过期与清理

### 9.1 缓存过期策略

| 缓存类型 | 过期时间 | 说明 |
|----------|----------|------|
| 商品缓存 | 永久 (0) | 由管理员操作更新，不自动过期 |
| 首页业务缓存 | 永久 | 实时监听保证最新 |
| 宝贝页业务缓存 | 永久 | 实时监听保证最新 |
| 购物车缓存 | 永久 | 实时监听保证最新 |
| 快递规则缓存 | 30 分钟 | 有过期时间 |
| 地址缓存 | 24 小时 | 有过期时间 |

### 9.2 缓存清理时机

| 场景 | 处理 |
|------|------|
| 用户主动刷新 | `wx.removeStorageSync('homeData')` |
| 商品删除 | 标记 `isDeleted: true`，不物理删除 |
| 小程序冷启动 | 内存缓存清空，本地存储保留 |

## 十、总结

### 10.1 架构优势

| 优势 | 说明 |
|------|------|
| **数据一致性** | 全局监听器保证所有页面看到的数据一致 |
| **性能优化** | 分层缓存减少数据库查询 |
| **实时性** | 数据变化时立即更新 UI |
| **职责分离** | 全局监听器负责缓存，页面监听器负责 UI |
| **资源节约** | 页面隐藏时关闭监听器，节省资源 |

### 10.2 关键设计决策

| 决策 | 原因 |
|------|------|
| **全局监听器永久运行** | 保证缓存始终最新 |
| **页面监听器按需启停** | 节省资源，只监听用户可见的页面 |
| **业务缓存永久存储** | 实时监听保证数据不会过期 |
| **深拷贝读取缓存** | 兼容真机只读对象的限制 |
| **init 快照跳过** | 避免重复更新，用户体验更流畅 |

### 10.3 监控与调试

| 日志标签 | 说明 |
|----------|------|
| `[全局监听器]` | 全局监听器相关日志 |
| `[首页]` / `[宝贝页面]` 等 | 各页面相关日志 |
| `首页-实时监听开启/关闭` | 监听器生命周期日志 |
