## 2026-04-17

### 1. 聊天页面表情/更多面板与列表同步问题 ✅

**问题描述**：点击表情或更多按钮时，面板弹出、输入框上移、消息列表滚动三者不同步，出现先后错位感。

**根本原因**：
- 之前的实现是两次渲染：先 `setData` 改面板状态，在 callback 里再调 `updateListBottomSpace()`，导致面板展开和列表上移分属不同帧。
- 曾尝试给面板加 CSS `animation: panelSlideUp 0.28s ease-out` 让面板有滑入动画，但反而造成"输入框和面板一起上移、列表有滑动动画"的新不协调。

**最终方案**：
1. 所有面板切换函数（`toggleEmojiPanel`、`toggleMorePanel`、`onInputFocus`、`onInputTap`、`handleOutsideTap`、`closePanelsOnLongPress`、`chooseImage`）改为**单次 setData**，同时写入面板状态 + `listBottomSpaceRpx` + `scrollTop`，保证同帧渲染。
2. `scroll-view` 保持 `scroll-with-animation="false"`，列表无滚动动画，瞬间到位。
3. 面板不加任何 CSS 动画，通过 `wx:if` 直接渲染瞬间出现。

**效果**：面板、输入框工具栏、消息列表三者完全在同一帧内同步出现/移动，无错位感。

**涉及文件**：
- `miniprogram/pages/message/service/index.js`：6 处面板切换函数改为单次原子 setData
- `miniprogram/pages/message/service/index.wxss`：去掉 `panelSlideUp` 动画及 `@keyframes` 定义

### 2. 客服会话卡片未读数量显示 ✅

**问题描述**：客服侧的会话卡片没有显示未读数量，用户侧也没有显示。

**根本原因**：
- 数据库设计已经正确（`unreadCountUser` 和 `unreadCountCustomerService` 两个字段）
- 前端 `loadAllSessions()` 只是简单地 `session.unreadCount = 0`，没有读取正确的字段
- 云函数 `sendMessage` 在发送消息时没有增加接收方的未读计数

**修复方案**：

**前端 (`miniprogram/pages/message/index.js`)**：
```javascript
// 根据身份获取正确的未读数量
const isCustomerService = this.data.isCustomerService;
session.unreadCount = isCustomerService 
  ? (session.unreadCountCustomerService || 0)
  : (session.unreadCountUser || 0);
```

**云函数 (`cloudfunctions/sendMessage/index.js`)**：
```javascript
// 获取会话信息以确定接收方
const sessionRes = await db.collection('sessions').doc(sessionId).get();
const session = sessionRes.data;

// 根据发送者身份增加接收方的未读数
if (role === 'customer_service') {
  // 客服发送消息，增加用户的未读计数
  updateData.unreadCountUser = (session.unreadCountUser || 0) + 1;
} else if (role === 'user') {
  // 用户发送消息，增加客服的未读计数
  updateData.unreadCountCustomerService = (session.unreadCountCustomerService || 0) + 1;
}
```

**效果**：会话卡片正确显示未读数量，参考通知卡片的红色徽章样式。

**涉及文件**：
- `miniprogram/pages/message/index.js`：修改 `loadAllSessions()` 的未读数初始化逻辑
- `cloudfunctions/sendMessage/index.js`：发送消息时增加接收方未读计数

### 3. 图片预览顺序和页码指示器 ✅

**问题描述**：点击某张图片打开预览，显示正确的图片，但马上跳到最久的照片，页码显示从 1 开始而不是当前图片的位置。

**根本原因**：
1. `openMediaPreview()` 初始设置正确的索引
2. 然后异步调用 `loadSessionMediaForPreview()` 从数据库中查询所有媒体
3. DB 查询使用 `orderBy('createTime', 'asc')` 按时间从早到新排序
4. 完成后直接替换 `previewMediaList` 和 `previewMediaIndex`，导致 swiper 跳转到最早的照片

**修复方案**：
- 改为 `orderBy('createTime', 'desc')`，让最新的图片在前
- 异步加载完成后重新计算 targetId 或 targetSrc 在新列表中的索引
- 保持初始打开的图片始终可见

**效果**：最新的图片排在前，预览列表从新到早逆序排列，打开预览时正确显示点击的图片，页码指示器准确。

**涉及文件**：
- `miniprogram/pages/message/service/index.js`：`loadSessionMediaForPreview()` 中改 `orderBy('createTime', 'desc')`
