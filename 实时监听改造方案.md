## 项目概述

### 核心问题

小程序当前存在两个严重影响用户体验的问题：

**问题一：数据不能自动更新**

当后台数据发生变化时（如商品价格调整、库存变动、订单状态更新），用户看到的页面数据是旧的，必须手动下拉刷新或重新进入页面才能看到最新内容。这在以下场景中体验尤其差：
- 商品列表/详情页：管理员修改了商品信息，用户看到的还是旧数据
- 订单列表/详情页：订单状态变更后，用户不知道
- 售后页面：售后处理进度无法实时感知
- 消息页面：新消息需要刷新才出现（已部分解决）

**问题二：刷新会导致丢失滚动位置**

用户在商品列表滚动到第 N 条，进入详情查看后返回，列表会重新加载并回到顶部，滚动位置丢失。这是因为 `onShow` 触发重新请求数据导致的。实时监听改造后，数据由 watch 推送增量更新，不需要重新拉取全量数据，自然不会破坏滚动位置。

### 改造目标

| 目标 | 具体效果 |
|------|---------|
| 数据自动更新 | 数据变更时页面自动刷新对应部分，无需用户手动操作 |
| 保持滚动位置 | 返回列表页时不重新拉取数据，滚动位置不丢失 |
| 快速首屏展示 | 进入页面时先展示缓存数据（极速），再由 watch 推送最新数据覆盖 |
| 降低请求压力 | 减少 `onShow` 中的全量查询，改为 watch 增量推送 |

### 缓存的持久性说明

微信小程序的「离开」有两种情况，行为不同：

| 场景 | 内存缓存 | 本地存储缓存 | 说明 |
|------|---------|------------|------|
| 用户切到其他 APP / 微信首页 | **保留**（小程序在后台） | 保留 | 重新回来时内存缓存仍可用 |
| 微信被系统杀进程 / 用户手动关闭小程序 | **丢失** | **保留** | 冷启动时走本地存储缓存 |
| 用户清除微信缓存 | 丢失 | **丢失** | 完全冷启动，重新请求数据 |

**结论**：本地存储缓存（`wx.setStorageSync`）在小程序被关闭后依然存在，只有用户主动清除微信数据才会丢失。正常冷启动的流程是：

```
APP 重新打开
  └─ 内存缓存为空（JS 进程重启）
     └─ 读本地存储缓存（wx.getStorageSync）→ 立即展示上次的数据
        └─ watch 启动 → 数据有变化时推送最新内容更新页面
```

所以即使用户关闭小程序后再重新打开，只要本地存储 TTL 没过期，仍然能看到上次的数据，不会白屏等待。

### 数据加载策略（缓存 + 实时监听组合）

这是本次改造的核心模式，每个页面都应遵循：

```
进入页面
  ↓
① 立即读取本地缓存 → 极速展示旧数据（用户感知无白屏）
  ↓
② 启动 watch 监听 → 数据库有变更时自动推送最新数据到页面
  ↓
③ watch 推送数据 → 更新页面 + 更新本地缓存
```

- **① 缓存读取**：使用 `SmartCache`，内存缓存（5分钟）+ 本地存储（2小时），进入页面时优先展示
- **② watch 监听**：页面显示（onShow）时启动，隐藏（onHide）时停止，卸载（onUnload）时销毁
- **③ 增量更新**：watch 推送的是变化的文档，只更新列表中对应条目，不重新渲染整个列表，不影响滚动位置

### 当前现状
- 使用腾讯云开发数据库
- 大多数列表页在 `onShow` 中全量重新拉取数据（导致滚动位置丢失）
- 缺乏本地缓存，每次进入页面都有白屏/加载等待
- 消息页面已有实时监听（`watch`），但游离在统一管理体系之外

### 技术栈
- **数据库**: 腾讯云数据库（支持变更流监听）
- **实时能力**: `db.collection().watch()` 云数据库变更流
- **缓存层**: `SmartCache`（内存缓存 + 本地存储二级缓存）
- **监听管理**: `watcherManager`（统一生命周期管理）+ `WatcherMixin`（页面自动管理）
- **状态管理**: 全局应用状态 + 页面本地状态

## 统一监听管理方案

### 1.1 统一监听管理类 (WatcherManager)

**文件**: `miniprogram/utils/watcher-manager.js`

```javascript
/**
 * 统一监听管理类
 * 提供页面级和全局级监听的统一管理
 */
class WatcherManager {
  constructor() {
    this.watchers = {};        // 页面级监听
    this.globalWatchers = [];  // 全局级监听
  }

  /**
   * 创建页面级监听
   * @param {string} key - 监听唯一标识
   * @param {Function} creator - 监听创建函数
   */
  create(key, creator) {
    // 避免重复创建
    if (this.watchers[key]) {
      console.warn(`[WatcherManager] Watcher "${key}" already exists`);
      return this.watchers[key];
    }

    const watcher = creator();
    const wrapper = {
      watcher,
      key,
      active: true,
      close: () => {
        if (wrapper.active && wrapper.watcher) {
          wrapper.watcher.close();
          wrapper.active = false;
          delete this.watchers[key];
        }
      }
    };

    this.watchers[key] = wrapper;
    console.log(`[WatcherManager] Created watcher: ${key}`);
    return wrapper;
  }

  /**
   * 获取监听
   */
  get(key) {
    return this.watchers[key];
  }

  /**
   * 销毁监听
   */
  destroy(key) {
    const wrapper = this.watchers[key];
    if (wrapper) {
      wrapper.close();
    }
  }

  /**
   * 销毁所有页面级监听
   */
  destroyAll() {
    Object.keys(this.watchers).forEach(key => {
      this.destroy(key);
    });
    console.log('[WatcherManager] Destroyed all watchers');
  }

  /**
   * 创建全局监听（登录后启动，退出登录停止）
   */
  createGlobal(creator) {
    const watcher = creator();
    this.globalWatchers.push(watcher);
    console.log('[WatcherManager] Created global watcher');
    return watcher;
  }

  /**
   * 销毁所有全局监听
   */
  destroyGlobals() {
    this.globalWatchers.forEach(watcher => {
      if (watcher && typeof watcher.close === 'function') {
        watcher.close();
      }
    });
    this.globalWatchers = [];
    console.log('[WatcherManager] Destroyed all global watchers');
  }
}

const watcherManager = new WatcherManager();
export default watcherManager;
```

### 1.2 页面级监听 Mixin（自动生命周期管理）

**文件**: `miniprogram/utils/watcher-mixin.js`

```javascript
/**
 * 监听生命周期 Mixin
 * 在 Page 中使用此 mixin，自动管理监听的启停
 * 
 * 使用方式：
 * import WatcherMixin from './watcher-mixin';
 * 
 * Page({
 *   ...WatcherMixin,
 *   onLoad() {
 *     this.addWatcher(() => db.collection('...').watch(...));
 *   }
 * })
 */

export const WatcherMixin = {
  data() {
    return {
      watchers: []
    };
  },

  onShow() {
    // 页面显示时启动所有监听
    this.startWatchers();
  },

  onHide() {
    // 页面隐藏时停止所有监听
    this.stopWatchers();
  },

  onUnload() {
    // 页面卸载时销毁所有监听
    this.destroyWatchers();
  },

  /**
   * 启动所有监听
   */
  startWatchers() {
    if (!this.data.watchers) return;
    
    this.data.watchers.forEach(wrapper => {
      if (!wrapper.active) {
        wrapper.watcher = wrapper.creator();
        wrapper.active = true;
      }
    });
    
    console.log(`[WatcherMixin] Started ${this.data.watchers.length} watchers`);
  },

  /**
   * 停止所有监听（保持监听对象，下次启动时重用）
   */
  stopWatchers() {
    if (!this.data.watchers) return;
    
    this.data.watchers.forEach(wrapper => {
      if (wrapper.active && wrapper.watcher) {
        wrapper.watcher.close();
        wrapper.active = false;
      }
    });
    
    console.log(`[WatcherMixin] Stopped ${this.data.watchers.length} watchers`);
  },

  /**
   * 销毁所有监听（清空监听对象）
   */
  destroyWatchers() {
    this.stopWatchers();
    this.setData({ watchers: [] });
  },

  /**
   * 添加监听
   * @param {Function} creator - 监听创建函数
   */
  addWatcher(creator) {
    const watchers = this.data.watchers || [];
    watchers.push({
      creator,
      watcher: null,
      active: false
    });
    this.setData({ watchers });
  }
};
```

### 1.3 使用示例

```javascript
import WatcherMixin from '../../utils/watcher-mixin';

Page({
  ...WatcherMixin,

  data: {
    orders: [],
    selectedStatus: 'all'
  },

  onLoad() {
    // 创建订单监听
    this.addWatcher(() => {
      return db.collection('orders')
        .where({ openid: wx.getStorageSync('openid') })
        .watch({
          onChange: (snapshot) => {
            this.setData({ orders: snapshot.docs });
          },
          onError: (err) => {
            console.error('订单监听失败:', err);
          }
        });
    });

    this.loadInitialOrders();
  },

  async loadInitialOrders() {
    const db = wx.cloud.database();
    const res = await db.collection('orders')
      .where({ openid: wx.getStorageSync('openid') })
      .get();
    this.setData({ orders: res.data });
  }
});
```

## 缓存设计方案

### 缓存更新机制说明

在理解具体配置之前，先明确缓存的工作原理：

**缓存在整个流程中的角色：**

```
冷启动（APP重开 / 长时间不用）
  └─ 读本地存储缓存 → 立刻展示（可能是几小时前的数据）
     └─ watch 启动 → 有变化时推送 → 更新页面 + 刷新缓存

热启动（同一会话内切换页面）
  └─ 读内存缓存 → 立刻展示（基本是最新数据，因为 watch 一直在更新它）
     └─ watch 已运行 → 有变化时继续推送 → 继续更新
```

**结论：内存缓存 TTL 不是关键**，因为 watch 运行期间会持续刷新内存缓存，TTL 失效前数据早就被 watch 更新了。**本地存储 TTL 才是关键**，它决定冷启动时允许展示多旧的数据。

### 缓存更新方式（针对性增量更新）

watch 推送变化时，**页面数据是增量更新的**（只改变那一条），**缓存写入是写整个更新后的数组**。两者结合既高效又正确，不需要重新查询数据库：

```
watch 推送：商品 A 价格从 99 → 89
  ↓
页面代码：找到 this.data.products 中 _id === A 的那条，只更新它的 price 字段
  ↓
setData({ products: updatedProducts })   // 只有 1 条数据变了，渲染高效
  ↓
productListCache.set(updatedProducts)    // 写入缓存（数组已是增量更新后的结果）
```

这是 **"增量更新页面数据 + 全量写入缓存"** 的组合，不需要给 SmartCache 增加 `patchItem()` 等复杂方法。

### 缓存实例清单

每种数据对应一个 SmartCache 实例，统一在 `utils/cacheInstances.js` 中定义和导出：

| 缓存 key | 对应数据 | 内存 TTL | 本地存储 TTL | 说明 |
|---------|---------|---------|------------|------|
| `cache_products_list` | 商品列表 | 30 分钟 | 24 小时 | 商品信息更新不频繁 |
| `cache_product_detail_{id}` | 商品详情（按 id） | 30 分钟 | 24 小时 | 每个商品单独缓存 |
| `cache_categories` | 商品分类 | 1 小时 | 72 小时 | 极少变化 |
| `cache_orders_list_{status}` | 订单列表（按状态） | 30 分钟 | 8 小时 | 按状态分开缓存 |
| `cache_order_detail_{id}` | 订单详情（按 id） | 30 分钟 | 8 小时 | 每个订单单独缓存 |
| `cache_sessions_list` | 会话列表 | 30 分钟 | 4 小时 | 消息较频繁，存储 TTL 短一些 |
| `cache_after_sales_list` | 售后列表 | 30 分钟 | 8 小时 | - |
| `cache_after_sales_detail_{id}` | 售后详情（按 id） | 30 分钟 | 8 小时 | - |
| `cache_settings` | 系统配置 | 1 小时 | 72 小时 | 几乎不变 |

> **说明**：内存 TTL 统一设为 30 分钟，watch 运行期间会持续刷新内存缓存，TTL 实际上很少触发过期判断。重要的是本地存储 TTL，它控制冷启动时的数据新鲜度。

### 各缓存 TTL 设置依据

| 数据类型 | 变化频率 | 展示旧数据的影响 | 本地存储 TTL |
|---------|---------|----------------|------------|
| 商品列表/详情 | 中（管理员手动修改） | 低（用户看到旧价格，但 watch 会很快更正） | 24 小时 |
| 订单列表/详情 | 中（状态自动流转） | 中（状态不对，但 watch 会更正） | 8 小时 |
| 会话列表 | 高（随时有新消息） | 低（会快速被 watch 更新） | 4 小时 |
| 售后列表/详情 | 低-中 | 中 | 8 小时 |
| 商品分类/系统配置 | 极低 | 极低 | 72 小时 |

### 缓存失效场景

以下情况需要主动清除对应缓存（不等 TTL 到期）：

| 触发事件 | 需要清除的缓存 | 原因 |
|---------|-------------|------|
| 用户退出登录 | 所有缓存 | 换账号后不能展示上一个账号的数据 |
| 用户主动下拉刷新 | 当前页面对应缓存 | 明确要求看最新数据 |
| 管理员提交了修改操作 | 对应数据的缓存 | 本端修改后缓存立即过期，避免展示旧值 |

```javascript
}

// ---- 会话/消息相关 ----
export const sessionListCache = new SmartCache('cache_sessions_list', {
  memoryCacheTTL: 30 * 60 * 1000,

});

// ---- 售后相关 ----

  memoryCacheTTL: 30 * 60 * 1000,

  return new SmartCache(`cache_after_sales_detail_${id}`, {

// ---- 系统配置 ----
export const settingsCache = new SmartCache('cache_settings', {
  memoryCacheTTL: 60 * 60 * 1000,
  storageCacheTTL: 72 * 60 * 60 * 1000

// ---- 退出登录时清除所有用户相关缓存 ----
export function clearAllUserCache() {
  productListCache.clear();
  categoryCache.clear();
  sessionListCache.clear();
  afterSalesListCache.clear();
  // 注意：按 id 的详情缓存用 wx.clearStorageSync() 或遍历清除
  try {
    const keys = wx.getStorageInfoSync().keys;
    keys.forEach(key => {
      if (key.startsWith('cache_')) {
        wx.removeStorageSync(key);
      }
    });
  } catch (e) {}
}
```

### 页面使用范式

```javascript
// pages/product-list/index.js
import { productListCache } from '../../utils/cacheInstances';
import watcherManager from '../../utils/watcherManager';

Page({
  onLoad() {
    // ① 先展示缓存
    const cached = productListCache.get();
    if (cached) {
      this.setData({ products: cached });
    }

    // ② 启动 watch，有变化时增量更新页面 + 刷新缓存
    this._startWatch();
  },

  _startWatch() {
    watcherManager.create('product_list', () => {
      return db.collection('products').watch({
        onChange: (snapshot) => {
          let products = [...this.data.products];

          snapshot.docChanges.forEach(change => {
            const doc = change.doc;
            const idx = products.findIndex(p => p._id === doc._id);

            if (change.dataType === 'add' && idx === -1) {
              products.unshift(doc);                   // 新增：插入头部
            } else if (change.dataType === 'modify' && idx !== -1) {
              products[idx] = { ...products[idx], ...doc };  // 修改：针对性更新
            } else if (change.dataType === 'remove' && idx !== -1) {
              products.splice(idx, 1);                 // 删除：移除
            }
          });

          this.setData({ products });
          productListCache.set(products);  // 更新缓存（写入增量更新后的完整数组）
        },
        onError: (err) => {
          console.error('商品监听失败:', err);
          setTimeout(() => {
            watcherManager.destroy('product_list');
            if (this.data.pageVisible) this._startWatch();
          }, 3000);
        }
      });
    });
  },

  onShow() {
    this.setData({ pageVisible: true });
    // onShow 中先 destroy 再重建，保证监听是新鲜的
    watcherManager.destroy('product_list');
    this._startWatch();
  },
  onHide()   { this.setData({ pageVisible: false }); watcherManager.destroy('product_list'); },
  onUnload() { watcherManager.destroy('product_list'); }
});
```

### 可扩展性：增量引入新场景不影响原有设计

缓存和实时监听的设计天然支持增量引入，原理如下：

**缓存层（SmartCache）：完全独立，互不干扰**

每个 SmartCache 实例通过 `key` 隔离，不同页面的缓存之间没有任何耦合。新增一个页面的缓存，只需在 `cacheInstances.js` 加一个新的导出，对已有代码零影响：

```javascript
// 原有缓存不动
export const productListCache = new SmartCache('cache_products_list', { ... });
export const sessionListCache = new SmartCache('cache_sessions_list', { ... });

// 新增：购物车缓存 ← 只需加这一行，其他完全不受影响
export const cartCache = new SmartCache('cache_cart', {
  memoryCacheTTL: 30 * 60 * 1000,
  storageCacheTTL: 2 * 60 * 60 * 1000
});
```

**监听层（watcherManager）：key 隔离，无全局副作用**

watcherManager 用 `Map` 存储所有监听，每个 key 完全独立。新增一个页面的监听，不会影响其他页面的监听状态：

```javascript
// 已有页面在运行：'msg_sessions'、'msg_chat'
watcherManager.getStats(); // => { pageWatchers: 2 }

// 新增购物车页面监听 ← 只需在新页面的 onShow 里加这一句
watcherManager.create('cart_watch', () => db.collection('cart').watch({ ... }));

// 已有监听完全不受影响
watcherManager.getStats(); // => { pageWatchers: 3 }
```

**总结：新增场景的完整步骤**

1. 在 `cacheInstances.js` 新增一个 `export const xxxCache = new SmartCache(...)`
2. 在新页面 `onLoad` 中读缓存展示
3. 在新页面 `onShow` 中 `watcherManager.create(唯一key, ...)`
4. 在 `onChange` 中增量更新页面数据并调用 `xxxCache.set()`
5. 在 `onHide/onUnload` 中 `watcherManager.destroy(key)`

不需要修改任何已有文件（除了 `cacheInstances.js` 加一行导出），对原有功能零风险。

```javascript
/**
 * 智能缓存类
 * 支持内存缓存 + 本地存储缓存的二级缓存策略
 */
class SmartCache {
  constructor(key, options = {}) {
    this.key = key;
    this.memoryCache = null;           // 内存缓存
    this.memoryTime = 0;               // 内存缓存时间戳
    this.memoryCacheTTL = options.memoryCacheTTL || 5 * 60 * 1000;  // 5分钟
    this.storageCacheTTL = options.storageCacheTTL || 2 * 60 * 60 * 1000; // 2小时
  }

  /**
   * 获取缓存数据
   */
  get() {
    // 1. 先查内存缓存（最快，当前会话有效）
    if (this.memoryCache && Date.now() - this.memoryTime < this.memoryCacheTTL) {
      console.log(`[SmartCache] Hit memory cache: ${this.key}`);
      return this.memoryCache;
    }

    // 2. 再查本地存储缓存
    try {
      const cached = wx.getStorageSync(this.key);
      if (cached) {
        const data = JSON.parse(cached);
        if (Date.now() - data.timestamp < this.storageCacheTTL) {
          // 更新内存缓存
          this.memoryCache = data.value;
          this.memoryTime = Date.now();
          console.log(`[SmartCache] Hit storage cache: ${this.key}`);
          return data.value;
        }
      }
    } catch (e) {
      console.error(`[SmartCache] Failed to read cache: ${this.key}`, e);
    }

    return null;
  }

  /**
   * 设置缓存数据
   */
  set(value) {
    // 更新内存缓存
    this.memoryCache = value;
    this.memoryTime = Date.now();

    // 更新本地存储缓存
    try {
      wx.setStorageSync(this.key, JSON.stringify({
        value,
        timestamp: Date.now()
      }));
      console.log(`[SmartCache] Cache set: ${this.key}`);
    } catch (e) {
      console.error(`[SmartCache] Failed to write cache: ${this.key}`, e);
    }
  }

  /**
   * 清除缓存
   */
  clear() {
    this.memoryCache = null;
    this.memoryTime = 0;
    try {
      wx.removeStorageSync(this.key);
    } catch (e) {
      console.error(`[SmartCache] Failed to clear cache: ${this.key}`, e);
    }
  }

  /**
   * 检查缓存是否过期
   */
  isExpired() {
    const cache = this.get();
    return !cache;
  }
}

export default SmartCache;
```

```javascript
/**
 * 启动监听
 * @param {string} key - 监听标识符
 * @param {Object} whereQuery - 查询条件
 * @param {Function} callback - 数据变化回调
 * @param {Object} options - 监听选项
 * @returns {Object} 返回监听实例
 */
function watch(key, whereQuery, callback, options = {}) {
  // 检查是否已存在相同的监听
  if (listeners.has(key)) {
    console.warn(`[RealTimeListener] 监听 ${key} 已存在，请先调用 unwatch 卸载`);
    return listeners.get(key);
  }

  const config = { ...listenerConfigs[key], ...options };
  const collectionName = config.collectionName;
  
  console.log(`[RealTimeListener] 启动监听: ${key}, 集合: ${collectionName}`);

  let query = db.collection(collectionName);
  
  // 应用查询条件
  if (whereQuery && Object.keys(whereQuery).length > 0) {
    Object.entries(whereQuery).forEach(([field, value]) => {
      query = query.where({ [field]: value });
    });
  }

  // 初始化状态跟踪
  const listenerState = {
    key,
    collectionName,
    callback,
    config,
    isActive: true,
    connectionStatus: 'connecting',
    reconnectCount: 0,
    lastUpdateTime: Date.now(),
    totalUpdates: 0,
    errorCount: 0,
    dedupeBuffer: new Map(), // 去重缓冲区
    reconnectTimer: null,
    watchInstance: null
  };

  // 启动真实监听
  startWatchInstance(listenerState);

  listeners.set(key, listenerState);
  return listenerState;
}

/**
 * 启动 watch 实例
 */
function startWatchInstance(listenerState) {
  const { key, collectionName, callback, config } = listenerState;
  
  try {
    let query = db.collection(collectionName);

    // 构建查询
    // 注意: watch 前的查询条件需要通过 where 链式调用
    // 这里简化处理，实际应用需根据具体查询条件动态构建

    const watchInstance = query.watch({
      onChange: (snapshot) => {
        handleWatchChange(listenerState, snapshot);
      },
      onError: (error) => {
        handleWatchError(listenerState, error);
      }
    });

    listenerState.watchInstance = watchInstance;
    listenerState.connectionStatus = 'connected';
    console.log(`[RealTimeListener] ${key} 连接成功`);
  } catch (error) {
    console.error(`[RealTimeListener] ${key} 启动失败:`, error);
    handleWatchError(listenerState, error);
  }
}

/**
 * 处理 watch 变化
 */
function handleWatchChange(listenerState, snapshot) {
  const { key, callback, config, dedupeBuffer } = listenerState;
  
  if (!snapshot.docChanges || snapshot.docChanges.length === 0) {
    return;
  }

  listenerState.lastUpdateTime = Date.now();
  listenerState.totalUpdates++;

  // 处理变化
  const changes = {
    added: [],
    modified: [],
    removed: []
  };

  snapshot.docChanges.forEach((change) => {
    const doc = change.doc || {};
    const dedupeKey = config.dedupeKey || '_id';
    const docId = doc[dedupeKey];

    // 去重逻辑：检查是否已处理过
    const cacheKey = `${key}_${change.dataType}_${docId}`;
    if (dedupeBuffer.has(cacheKey)) {
      const { timestamp, revision } = dedupeBuffer.get(cacheKey);
      // 如果 5 秒内重复，跳过
      if (Date.now() - timestamp < 5000 && doc._rev === revision) {
        return;
      }
    }

    // 更新去重缓冲
    dedupeBuffer.set(cacheKey, {
      timestamp: Date.now(),
      revision: doc._rev
    });

    // 分类处理
    switch (change.dataType) {
      case 'add':
        changes.added.push(doc);
        break;
      case 'modify':
        changes.modified.push(doc);
        break;
      case 'remove':
        changes.removed.push(doc);
        break;
    }
  });

  // 调用用户回调
  if (callback && typeof callback === 'function') {
    callback(changes, {
      timestamp: Date.now(),
      snapshotKey: snapshot.snapshotKey,
      listenerKey: key
    });
  }

  // 清理过期去重缓冲（防止内存泄漏）
  if (dedupeBuffer.size > 1000) {
    const now = Date.now();
    for (const [k, v] of dedupeBuffer.entries()) {
      if (now - v.timestamp > 10000) {
        dedupeBuffer.delete(k);
      }
    }
  }

  listenerState.connectionStatus = 'connected';
}

/**
 * 处理 watch 错误
 */
function handleWatchError(listenerState, error) {
  const { key, config } = listenerState;
  
  listenerState.errorCount++;
  listenerState.connectionStatus = 'error';
  
  console.error(`[RealTimeListener] ${key} 监听出错:`, error);

  // 自动重连逻辑
  if (config.autoReconnect && listenerState.reconnectCount < config.maxReconnectAttempts) {
    const delay = config.reconnectDelay * Math.pow(2, listenerState.reconnectCount);
    listenerState.reconnectTimer = setTimeout(() => {
      listenerState.reconnectCount++;
      console.log(`[RealTimeListener] ${key} 第 ${listenerState.reconnectCount} 次重连...`);
      startWatchInstance(listenerState);
    }, delay);
  } else if (listenerState.reconnectCount >= config.maxReconnectAttempts) {
    console.error(`[RealTimeListener] ${key} 重连次数已达上限，停止重连`);
    listenerState.connectionStatus = 'failed';
  }
}

/**
 * 卸载监听
 * @param {string} key - 监听标识符
 */
function unwatch(key) {
  const listenerState = listeners.get(key);
  
  if (!listenerState) {
    console.warn(`[RealTimeListener] 监听 ${key} 不存在`);
    return;
  }

  console.log(`[RealTimeListener] 卸载监听: ${key}`);

  // 关闭 watch
  if (listenerState.watchInstance && typeof listenerState.watchInstance.close === 'function') {
    try {
      listenerState.watchInstance.close();
    } catch (error) {
      console.error(`[RealTimeListener] ${key} 关闭失败:`, error);
    }
  }

  // 清理重连定时器
  if (listenerState.reconnectTimer) {
    clearTimeout(listenerState.reconnectTimer);
  }

  // 清理缓冲区
  listenerState.dedupeBuffer.clear();
  
  listenerState.isActive = false;
  listeners.delete(key);
}

/**
 * 卸载所有监听
 */
function unwatchAll() {
  const keys = Array.from(listeners.keys());
  keys.forEach(key => unwatch(key));
  console.log(`[RealTimeListener] 已卸载所有 ${keys.length} 个监听`);
}

/**
 * 获取监听状态
 * @param {string} key - 监听标识符
 */
function getStatus(key) {
  const listener = listeners.get(key);
  if (!listener) {
    return null;
  }

  return {
    key: listener.key,
    status: listener.connectionStatus,
    isActive: listener.isActive,
    updateCount: listener.totalUpdates,
    errorCount: listener.errorCount,
    reconnectCount: listener.reconnectCount,
    lastUpdateTime: listener.lastUpdateTime
  };
}

/**
 * 获取所有监听状态
 */
function getAllStatus() {
  const status = {};
  listeners.forEach((listener, key) => {
    status[key] = getStatus(key);
  });
  return status;
}

export {
  watch,
  unwatch,
  unwatchAll,
  getStatus,
  getAllStatus,
  listenerConfigs
};
```

```javascript
/**
 * 订单列表实时监听适配层

import { watch, unwatch } from './realtimeListener';

const LISTENER_KEY = 'orders_watch';

export class OrderListener {
  constructor(page) {
    this.page = page;
    this.listenerKey = LISTENER_KEY;
    this.localCache = new Map(); // 本地缓存最新数据
  }

  /**
   * 启动订单监听
   * @param {Object} options - 监听选项
   */
  start(options = {}) {
    const whereQuery = this.buildWhereQuery(options);
    
    const callback = (changes, meta) => {
      this.handleOrderChanges(changes, meta);
    };

    watch(this.listenerKey, whereQuery, callback, {
      collectionName: 'orders',
      dedupeKey: '_id',
      maxReconnectAttempts: 5
    });
  }

  /**
   * 构建查询条件
   */
  buildWhereQuery(options = {}) {
    const openid = wx.getStorageSync('openid');
    const query = { openid };

    // 按状态筛选
    if (options.status && options.status !== 'all') {
      query.status = options.status;
    }

    return query;
  }

  /**
   * 处理订单变化
   */
  handleOrderChanges(changes, meta) {
    const { added, modified, removed } = changes;

    // 获取当前页面数据
    let { orders, originalOrders } = this.page.data;
    orders = orders || [];
    originalOrders = originalOrders || [];

    let hasChanges = false;

    // 处理新增订单
    if (added && added.length > 0) {
      added.forEach((order) => {
        // 检查是否已存在
        const existIndex = originalOrders.findIndex(o => o._id === order._id);
        if (existIndex === -1) {
          originalOrders.unshift(order);
          hasChanges = true;
        }
      });
    }

    // 处理修改订单
    if (modified && modified.length > 0) {
      modified.forEach((order) => {
        const existIndex = originalOrders.findIndex(o => o._id === order._id);
        if (existIndex !== -1) {
          // 合并数据（保留本地优先级高的字段）
          const oldOrder = originalOrders[existIndex];
          originalOrders[existIndex] = {
            ...oldOrder,
            ...order,
            // 保留本地字段
            localFlag: oldOrder.localFlag
          };
          hasChanges = true;
        }
      });
    }

    // 处理删除订单
    if (removed && removed.length > 0) {
      removed.forEach((order) => {
        const existIndex = originalOrders.findIndex(o => o._id === order._id);
        if (existIndex !== -1) {
          originalOrders.splice(existIndex, 1);
          hasChanges = true;
        }
      });
    }

    // 如果有变化，更新页面数据
    if (hasChanges) {
      this.page.setData({
        originalOrders,
        orders: this.applyFilterAndSort(originalOrders)
      });
    }
  }

  /**
   * 应用筛选和排序
   */
  applyFilterAndSort(orders) {
    const { selectedStatus, sortType } = this.page.data;
    
    let filtered = orders;

    // 按状态筛选
    if (selectedStatus && selectedStatus !== 'all') {
      filtered = orders.filter(o => o.status === selectedStatus);
    }

    // 按时间排序（最新的在前）
    filtered.sort((a, b) => {
      const timeA = new Date(a.createdAt).getTime();
      const timeB = new Date(b.createdAt).getTime();
      return timeB - timeA;
    });

    return filtered;
  }

  /**
   * 停止监听
   */
  stop() {
    unwatch(this.listenerKey);
  }
}
```

```javascript
      this.handleSessionChanges(changes, meta);
    };

    watch(this.sessionListenerKey, { userId: openid }, callback, {
      collectionName: 'sessions',
      dedupeKey: '_id',
      maxReconnectAttempts: 5
    });
  }

  /**
   * 处理消息变化
   */
  handleMessageChanges(changes, meta, sessionId) {
    const { added, modified, removed } = changes;
    const { messages, messageListReady } = this.page.data;

    // 如果页面未准备好，缓冲消息
    if (!messageListReady) {
      if (added && added.length > 0) {
        this.messageBuffer.push(...added);
      }
      return;
    }

    let hasChanges = false;
    let newMessages = [...(messages || [])];

    // 处理新增消息（关键：避免本人消息重复）
    if (added && added.length > 0) {
      added.forEach((msg) => {
        // 检查是否已存在（去重）
        const existIndex = newMessages.findIndex(m => m._id === msg._id);
        
        if (existIndex === -1) {
          // 检查是否是本人消息
          const isOwnMessage = msg.openid === this.page.data.openid;
          
          // 本人消息已在本地即时追加，跳过 watch 的重复
          if (!isOwnMessage) {
            newMessages.push(msg);
            hasChanges = true;
            
            // 清除未读标记（对端消息）
            this.clearUnreadCount(sessionId);
          }
        }
      });
    }

    // 处理消息修改（撤回、编辑等）
    if (modified && modified.length > 0) {
      modified.forEach((msg) => {
        const existIndex = newMessages.findIndex(m => m._id === msg._id);
        if (existIndex !== -1) {
          newMessages[existIndex] = {
            ...newMessages[existIndex],
            ...msg
          };
          hasChanges = true;
        }
      });
    }

    // 处理消息删除
    if (removed && removed.length > 0) {
      removed.forEach((msg) => {
        const existIndex = newMessages.findIndex(m => m._id === msg._id);
        if (existIndex !== -1) {
          newMessages.splice(existIndex, 1);
          hasChanges = true;
        }
      });
    }

    // 更新页面数据
    if (hasChanges) {
      this.page.setData({
        messages: newMessages,
        groupedMessages: this.page.groupMessages(newMessages)
      });

      // 滚动到底部显示最新消息
      this.scrollToBottom();
    }
  }

  /**
   * 处理会话变化
   */
  handleSessionChanges(changes, meta) {
    const { added, modified, removed } = changes;
    const { sessions } = this.page.data;

    let newSessions = [...(sessions || [])];
    let hasChanges = false;

    // 处理新增会话
    if (added && added.length > 0) {
      added.forEach((session) => {
        const existIndex = newSessions.findIndex(s => s._id === session._id);
        if (existIndex === -1) {
          newSessions.unshift(session);
          hasChanges = true;
        }
      });
    }

    // 处理会话修改（未读计数、最后消息等）
    if (modified && modified.length > 0) {
      modified.forEach((session) => {
        const existIndex = newSessions.findIndex(s => s._id === session._id);
        if (existIndex !== -1) {
          newSessions[existIndex] = {
            ...newSessions[existIndex],
            ...session
          };
          hasChanges = true;
        }
      });
    }

    // 处理会话删除
    if (removed && removed.length > 0) {
      removed.forEach((session) => {
        const existIndex = newSessions.findIndex(s => s._id === session._id);
        if (existIndex !== -1) {
          newSessions.splice(existIndex, 1);
          hasChanges = true;
        }
      });
    }

    // 更新页面数据
    if (hasChanges) {
      this.page.setData({ sessions: newSessions });
    }
  }

  /**
   * 滚动到底部
   */
  scrollToBottom() {
    this.page.setData({
      scrollTop: 999999
    });
  }

  /**
   * 清除未读计数
   */
  clearUnreadCount(sessionId) {
    // 异步调用云函数
    wx.cloud.callFunction({
      name: 'clearUnreadCount',
      data: { sessionId }
    }).catch(error => {
      console.error('清除未读计数失败:', error);
    });
  }

  /**
   * 处理缓冲的消息
   */
  flushMessageBuffer() {
    if (this.messageBuffer.length > 0) {
      const buffer = this.messageBuffer;
      this.messageBuffer = [];
      
      const changes = {
        added: buffer,
        modified: [],
        removed: []
      };

      this.handleMessageChanges(changes, {}, this.page.data.sessionId);
    }
  }

  /**
   * 停止消息监听
   */
  stopMessageListener() {
    unwatch(this.messageListenerKey);
  }

  /**
   * 停止会话监听
   */
  stopSessionListener() {
    unwatch(this.sessionListenerKey);
  }

  /**
   * 停止所有监听
   */
  stopAll() {
    this.stopMessageListener();
    this.stopSessionListener();
  }
}
```

```javascript
export class ConflictResolver {
  /**
   * 解决本地修改 vs 远端更新的冲突
   * 策略: 时间戳 + 操作优先级
   */
  static resolveOrderConflict(localOrder, remoteOrder) {
    // 如果本地有未上传的修改，优先保留本地
    if (localOrder._localModified && !localOrder._uploaded) {
      return localOrder;
    }

    // 否则使用更新时间最新的版本
    const localTime = new Date(localOrder.updatedAt).getTime();
    const remoteTime = new Date(remoteOrder.updatedAt).getTime();

    return remoteTime > localTime ? remoteOrder : localOrder;
  }

  /**
   * 解决消息顺序冲突
   * 策略: createTime + messageId
   */
  static resolveMessageOrder(msg1, msg2) {
    const time1 = msg1.createTime instanceof Date 
      ? msg1.createTime.getTime() 
      : new Date(msg1.createTime).getTime();
    
    const time2 = msg2.createTime instanceof Date 
      ? msg2.createTime.getTime() 
      : new Date(msg2.createTime).getTime();

    if (time1 === time2) {
      // 时间相同，按 ID 字典序排序
      return msg1._id.localeCompare(msg2._id);
    }

    return time1 - time2;
  }
}
```

```javascript
export class PaginatedWatcher {
  constructor(config = {}) {
    this.pageSize = config.pageSize || 50;
    this.maxPages = config.maxPages || 5; // 最多保留 5 页
    this.allData = [];
    this.currentPage = 0;
  }

  /**
   * 添加新数据
   */
  addData(items) {
    if (!Array.isArray(items)) return;

    items.forEach(item => {
      const existIndex = this.allData.findIndex(
        d => d._id === item._id
      );

      if (existIndex === -1) {
        this.allData.unshift(item);
      } else {
        this.allData[existIndex] = item;
      }
    });

    // 限制内存占用
    if (this.allData.length > this.pageSize * this.maxPages) {
      this.allData = this.allData.slice(
        0,
        this.pageSize * this.maxPages
      );
    }
  }

  /**
   * 获取当前页数据
   */
  getCurrentPage() {
    const start = this.currentPage * this.pageSize;
    const end = start + this.pageSize;
    return this.allData.slice(start, end);
  }

  /**
   * 下一页
   */
  nextPage() {
    const maxPages = Math.ceil(this.allData.length / this.pageSize);
    if (this.currentPage < maxPages - 1) {
      this.currentPage++;
      return this.getCurrentPage();
    }
    return [];
  }

  /**
   * 上一页
   */
  prevPage() {
    if (this.currentPage > 0) {
      this.currentPage--;
      return this.getCurrentPage();
    }
    return [];
  }
}
```

```javascript
export class SmartCache {
  constructor(maxSize = 100, ttl = 5 * 60 * 1000) {
    this.maxSize = maxSize;
    this.ttl = ttl;
    this.cache = new Map();
  }

  set(key, value) {
    // 超过最大容量，删除最旧的条目
    if (this.cache.size >= this.maxSize) {
      const firstKey = this.cache.keys().next().value;
      this.cache.delete(firstKey);
    }

    this.cache.set(key, {
      value,
      timestamp: Date.now()
    });
  }

  get(key) {
    const item = this.cache.get(key);
    
    if (!item) {
      return null;
    }

    // 检查是否过期
    if (Date.now() - item.timestamp > this.ttl) {
      this.cache.delete(key);
      return null;
    }

    // 更新时间戳（LRU）
    this.cache.delete(key);
    this.cache.set(key, item);

    return item.value;
  }

  clear() {
    this.cache.clear();
  }

  getStats() {
    return {
      size: this.cache.size,
      maxSize: this.maxSize,
      ttl: this.ttl
    };
  }
}
```

```javascript
export class ThrottleDebounce {
  /**
   * 防抖：合并多个快速触发的事件为一个
   */
  static debounce(fn, delay = 300) {
    let timer = null;

    return function(...args) {
      if (timer) {
        clearTimeout(timer);
      }

      timer = setTimeout(() => {
        fn.apply(this, args);
        timer = null;
      }, delay);
    };
  }

  /**
   * 节流：限制函数执行频率
   */
  static throttle(fn, interval = 300) {
    let lastTime = 0;

    return function(...args) {
      const now = Date.now();

      if (now - lastTime >= interval) {
        fn.apply(this, args);
        lastTime = now;
      }
    };
  }

  /**
   * 请求节流：同时只允许一个请求
   */
  static requestThrottle(fn) {
    let pending = false;

    return async function(...args) {
      if (pending) {
        return;
      }

      pending = true;
      try {
        return await fn.apply(this, args);
      } finally {
        pending = false;
      }
    };
  }
}
```

## 常见问题与解决方案

### Q1: 如何处理离线用户的数据一致性？

**A**: 采用以下策略：
- 离线时，本地修改标记为 `_localModified = true`
- 恢复联网时，先上传本地修改，再启动监听
- 使用冲突解决器处理本地修改 vs 远端更新的冲突

### Q2: 内存占用会不会持续增长？

**A**: 不会。实现了以下机制：
- 去重缓冲区自动清理过期条目
- 分页观察器限制最多缓存数据条数
- 智能缓存采用 LRU 策略，超过最大容量自动删除

### Q3: 如何监控监听健康状态？

**A**: 使用提供的监控方法：
```javascript
// 查询单个监听状态
const status = getStatus('orders_watch');
```

### Q4: 成本会增加吗？

**A**: 实际上会降低成本：
- 数据库查询次数大幅减少
- 减少了轮询造成的云函数调用

## 现有监听页面纳管方案

> 此方案专门针对**已有实时监听但游离在管理体系之外**的页面进行改造，将其纳入 `watcherManager` 统一管理。

### 现状摸排

全局搜索 `.watch()` 调用，当前小程序中共有 **2 个业务页面**自行管理监听：

| 页面 | 文件 | 监听变量 | 监听集合 | 当前问题 |
|------|------|---------|---------|---------|
| TabBar 会话列表 | `pages/message/index.js` | `this.sessionListener` | `sessions` | 无自动重连；游离在管理体系外 |
| 聊天详情 | `pages/message/service/index.js` | `this.messageWatch` | `messages` | 无自动重连；游离在管理体系外 |

### 改造原则

1. **业务逻辑不动**：`onChange` 回调内容一行不改，只改生命周期管理层
2. **保留刻意设计**：`service/index.js` 的 `onHide` 刻意不停止监听（切后台保持连接），改造后继续保留此行为
3. **补充自动重连**：在 `onError` 中加入 3s 后销毁并重新创建的逻辑
4. **统一 key 命名**：`msg_sessions`（会话列表）、`msg_chat`（聊天详情）

### 改造内容 - `pages/message/index.js`（会话列表）

**改动量：约 6 行**

| 位置 | 改造前 | 改造后 |
|------|--------|--------|
| 顶部 | 无 import | `import watcherManager from '../../utils/watcherManager'` |
| `listenSessions()` | `this.sessionListener = query.watch({...})` | `watcherManager.create('msg_sessions', () => query.watch({...}))` |
| `onError` 回调 | 只打印日志 | 打印日志 + 3s 后 `destroy('msg_sessions')` + 重新调用 `listenSessions()` |
| `onShow` | `this.sessionListener.close()` | `watcherManager.destroy('msg_sessions')` |
| `onHide` | `this.sessionListener.close(); this.sessionListener = null` | `watcherManager.destroy('msg_sessions')` |
| `onUnload` | `this.sessionListener.close(); this.sessionListener = null` | `watcherManager.destroy('msg_sessions')` |

**改造后关键代码示意：**

```javascript
import watcherManager from '../../utils/watcherManager';

// listenSessions() 中：
watcherManager.create('msg_sessions', () => {
  return query.watch({
    onChange: (snapshot) => {
      // ← 原有 onChange 逻辑完全不变
    },
    onError: (error) => {
      console.error('监听会话失败:', error);
      // 新增：3s 后自动重连
      setTimeout(() => {
        watcherManager.destroy('msg_sessions');
        if (this.data.pageVisible) {
          this.listenSessions();
        }
      }, 3000);
    }
  });
});

// onShow 中：
watcherManager.destroy('msg_sessions'); // 替换 this.sessionListener.close()
this.listenSessions();

// onHide / onUnload 中：
watcherManager.destroy('msg_sessions'); // 替换 this.sessionListener.close(); this.sessionListener = null
```

### 改造内容 - `pages/message/service/index.js`（聊天详情）

**改动量：约 4 行**

| 位置 | 改造前 | 改造后 |
|------|--------|--------|
| 顶部 | 无 import | `import watcherManager from '../../utils/watcherManager'` |
| `listenMessages()` | `this.messageWatch = db.collection(...).watch({...})` | `watcherManager.create('msg_chat', () => db.collection(...).watch({...}))` |
| `onError` 回调 | 只打印日志 | 打印日志 + 3s 后 `destroy('msg_chat')` + 重新调用 `listenMessages()` |
| `onUnload` | `if (this.messageWatch) { this.messageWatch.close() }` | `watcherManager.destroy('msg_chat')` |

> **注意**：`onHide` 刻意不调用 `destroy`，保留切换页面时保持连接的原有设计。

**改造后关键代码示意：**

```javascript
import watcherManager from '../../utils/watcherManager';

// listenMessages() 中：
watcherManager.create('msg_chat', () => {
  return db.collection('messages')
    .where({ sessionId })
    .watch({
      onChange: (snapshot) => {
        // ← 原有 onChange 逻辑完全不变（包含去重、滚动等）
      },
      onError: (error) => {
        console.error('监听消息失败', error);
        // 新增：3s 后自动重连
        setTimeout(() => {
          watcherManager.destroy('msg_chat');
          if (this.data.sessionId) {
            this.listenMessages();
          }
        }, 3000);
      }
    });
});

// onUnload 中：
watcherManager.destroy('msg_chat'); // 替换 if (this.messageWatch) { this.messageWatch.close() }
// onHide 不做任何改动（保持连接）
```

### 改造后效果

```javascript
// 在任何地方均可查看当前所有监听的健康状态
watcherManager.printStats();
// => [WatcherManager] Stats: { pageWatchers: 2, globalWatchers: 0, totalWatchers: 2 }
// 活跃监听 key：'msg_sessions'、'msg_chat'
```

### 风险评估

| 风险点 | 评级 | 应对 |
|--------|------|------|
| onChange 业务逻辑被改动 | 无风险 | 原则：onChange 内容一行不动 |
| `service` 页 onHide 不停止的设计被破坏 | 无风险 | onHide 不加 destroy，维持原行为 |
| 自动重连与手动重建冲突 | 低风险 | 重连前先 destroy 保证干净创建 |
| 两页面 key 冲突 | 无风险 | 不同 key：`msg_sessions` / `msg_chat` |
