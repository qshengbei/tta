# 消息系统设计

## 一、非客服消息

订单状态更新消息、系统消息、活动通知等。

### 1. 集合结构设计

#### 基本字段

| 字段名         | 类型      | 描述               | 示例                                                                                      |
| ----------- | ------- | ---------------- | --------------------------------------------------------------------------------------- |
| \_id        | String  | 消息ID（自动生成）       | "msg\_001"                                                                              |
| openid      | String  | 用户ID，关联用户集合      | "oABC123..."                                                                            |
| type        | String  | 消息类型             | "activity", "system", "order", "service", "logistics", "stock", "appointment", "review" |
| subType     | String  | 消息子类型            | "new\_product", "new\_collection", "promotion", "limited\_discount"                     |
| title       | String  | 消息标题             | "新品发布：春季系列"                                                                             |
| content     | String  | 消息内容             | "Touch the Aura 春季系列新品已上线，限时8折优惠"                                                       |
| status      | String  | 消息状态             | "unread", "read", "deleted"                                                             |
| createTime  | Date    | 创建时间             | ISODate("2026-03-26T08:00:00Z")                                                         |
| relatedId   | String  | 关联ID（如订单ID、活动ID） | "order\_001"                                                                            |
| relatedType | String  | 关联类型             | "order", "activity", "product"                                                          |
| extra       | Object  | 额外信息             | {"discount": "8折", "endTime": "2026-04-01"}                                             |
| isImportant | Boolean | 是否重要消息           | true/false                                                                              |
| isPinned    | Boolean | 是否置顶             | true/false                                                                              |

### 2. 消息类型设计

#### 1. 活动消息 (activity)

- new\_product（新品发布）
  - 内容：新品名称、图片、价格、上线时间、特点介绍
  - 示例："【新品发布】Touch the Aura 推出全新极简金属发夹系列，3月26日正式上线，限时8折优惠"
- new\_collection（新系列发布）
  - 内容：系列名称、主题、包含产品、上线时间、系列故事
  - 示例："【新系列发布】Touch the Aura 2026春季系列 'Lightness' 正式上线，以轻盈为主题，打造春日灵动造型"
- promotion（节日活动）
  - 内容：活动名称、活动时间、优惠内容、参与方式
  - 示例："【节日活动】3月女神节特别活动，全场满100减20，限时3天"
- limited\_discount（限时折扣）
  - 内容：折扣商品、折扣力度、开始结束时间
  - 示例："【限时折扣】今日限时秒杀，经典发圈系列5折起，仅剩2小时"
- member\_exclusive（会员专享）
  - 内容：会员等级、专享优惠、活动时间
  - 示例："【会员专享】VIP会员专享福利，全场商品额外9折，还有专属礼品"
- flash\_sale（闪购）
  - 内容：闪购商品、闪购价格、开始结束时间
  - 示例："【闪购】10点准时开抢，限量版发夹仅需99元，数量有限"
- pre\_order（预售）
  - 内容：预售商品、预售价格、发货时间
  - 示例："【预售】夏季新品预售开启，预付定金立减50元，4月15日发货"

#### 2. 系统消息 (system)

- account\_security（账户安全）
  - 内容：安全提醒、操作时间、建议措施
  - 示例："【账户安全】您的账户于3月26日10:00在新设备登录，如非本人操作，请及时修改密码"
- real\_name\_verify（实名认证）
  - 内容：认证结果、认证时间
  - 示例："【实名认证】您的实名认证已通过，感谢您的配合"
- password\_change（密码修改）
  - 内容：修改时间、安全提醒
  - 示例："【密码修改】您的账户密码已于3月26日10:30修改，请妥善保管"
- system\_maintenance（系统维护）
  - 内容：维护时间、影响范围
  - 示例："【系统维护】系统将于3月27日凌晨2:00-4:00进行维护，期间可能影响部分功能使用"
- version\_update（版本更新）
  - 内容：更新内容、更新时间
  - 示例："【版本更新】小程序已更新至v1.2.0版本，新增消息中心功能，优化用户体验"

#### 3. 订单状态更新消息 (order)

- order\_created（订单创建）
  - 内容：订单号、商品信息、订单金额

#### 4. 其他消息类型

  - 内容：商品名称、库存状态、补货时间

### 3. 集合索引设计

#### 推荐索引

1. openid + status + createTime：用于快速查询用户的消息列表，按状态和时间排序
2. openid + type + createTime：用于快速查询用户的特定类型消息
3. relatedId + relatedType：用于快速查询与特定对象相关的消息
4. createTime：用于按时间范围查询消息

### 4. 数据安全

#### 权限设置

- 读取权限：仅用户本人可读取自己的消息
- 写入权限：仅管理员和系统可写入消息
- 更新权限：用户可更新自己消息的状态（如标记已读）
- 删除权限：用户可删除自己的消息，管理员可删除所有消息

#### 数据脱敏

- 敏感信息（如订单号、物流单号）可进行部分脱敏处理
- 确保用户隐私信息不被泄露

### 5. 实施建议

1. **消息推送机制**：
   - 系统消息和重要消息通过微信模板消息推送
   - 活动消息和非紧急消息通过小程序消息中心展示
2. **消息存储策略**：
   - 保留最近3个月的消息
   - 超过3个月的消息可归档或删除
3. **消息通知频率**：
   - 避免过于频繁的消息推送，影响用户体验
   - 提供消息通知设置，允许用户自定义通知偏好
4. **消息互动**：
   - 支持消息回复功能，方便用户与客服沟通
   - 支持消息点赞、分享等互动功能
5. **数据分析**：
   - 分析消息阅读率、点击率等数据
   - 根据用户行为优化消息内容和推送策略

## 二、客服消息

### 1. 基于云开发的实现

#### 架构

- 云数据库：存储消息记录
- 云函数：处理消息发送和接收
- 小程序前端：用户消息界面

#### 实现步骤

1. 创建消息集合，存储用户和客服的消息
2. 实现消息发送功能，通过云函数将消息存储到数据库
3. 实现消息接收功能，通过云函数推送消息给用户
4. 开发客服工作台，查看和回复用户消息

#### 特点

- 完全自定义，可根据业务需求定制功能
- 需要更多的开发工作
- 可与其他系统集成

### 2. 技术架构

#### 1. 云数据库

- **消息集合**：存储用户和客服的聊天消息
- **会话集合**：存储用户与客服的会话信息
- **客服状态集合**：存储客服的在线状态和工作负载

#### 2. 云函数

- **发送消息**：处理消息发送逻辑
- **接收消息**：处理消息接收逻辑
- **实时推送**：利用云开发的实时数据库能力
- **客服分配**：处理客服分配逻辑

#### 3. 小程序前端

- **聊天界面**：显示消息列表
- **消息输入**：发送消息
- **实时更新**：监听消息变化
- **客服工作台**：管理会话和回复消息

#### 2. 云函数实现

##### 发送消息云函数 (sendMessage)

```javascript
// cloudfunctions/sendMessage/index.js
const cloud = require('wx-server-sdk');
cloud.init();
const db = cloud.database();

exports.main = async (event, context) => {
  const { sessionId, content, type = 'text' } = event;
  const { OPENID } = cloud.getWXContext();
  
  try {
    // 创建消息
    const message = {
      _id: `msg_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`,
      sessionId,
      openid: OPENID,
      role: 'user',
      content,
      type,
      createTime: new Date(),
      status: 'sent'
    };
    
    // 保存消息
    await db.collection('messages').add({ data: message });
    
    // 更新会话
    await db.collection('sessions').doc(sessionId).update({
      data: {
        lastMessage: message,
        lastMessageTime: new Date(),
        lastActiveTime: new Date(),
        unreadCount: db.command.inc(1)
      }
    });
    
    return {
      success: true,
      messageId: message._id
    };
  } catch (error) {
    console.error('发送消息失败', error);
    return {
      success: false,
      error: error.message
    };
  }
};
```

##### 分配客服云函数 (assignCustomerService)

```javascript
// cloudfunctions/assignCustomerService/index.js
const cloud = require('wx-server-sdk');
cloud.init();
const db = cloud.database();

exports.main = async (event, context) => {
  const { userId } = event;

  try {
    // 查询在线客服
    const onlineCS = await db.collection('customer_service_status')
      .where({ status: 'online' })
      .orderBy('activeSessions', 'asc')
      .get();

    if (onlineCS.data.length === 0) {
      return { success: false, error: "暂无在线客服" };
    }

    // 选择活跃会话数最少的客服
    const assignedCS = onlineCS.data[0];

    // 更新客服活跃会话数
    await db.collection('customer_service_status').doc(assignedCS._id).update({
      data: {
        activeSessions: db.command.inc(1)
      }
    });

    return { 
      success: true, 
      customerServiceId: assignedCS.customerServiceId,
      customerServiceName: assignedCS.name
    };
  } catch (error) {
    console.error('分配客服失败', error);
    return { success: false, error: error.message };
  }
};
```

##### 创建会话云函数 (createSession)

```javascript
// cloudfunctions/createSession/index.js
const cloud = require('wx-server-sdk');
cloud.init();
const db = cloud.database();

exports.main = async (event, context) => {
  const { userId, initialMessage } = event;

  try {
    // 分配客服
    const assignResult = await cloud.callFunction({
      name: 'assignCustomerService',
      data: { userId }
    });

    if (!assignResult.result.success) {
      return { success: false, error: assignResult.result.error };
    }

    const { customerServiceId, customerServiceName } = assignResult.result;

    // 创建会话
    const session = {
      _id: `session_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`,
      userId,
      customerServiceId,
      customerServiceName,
      assignedTo: customerServiceId,
      assignedTime: new Date(),
      lastMessage: {
        content: initialMessage,
        createTime: new Date()
      },
      lastMessageTime: new Date(),
      lastActiveTime: new Date(),
      unreadCount: 1,
      status: 'active'
    };

    await db.collection('sessions').add({ data: session });

    // 创建初始消息
    const message = {
      _id: `msg_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`,
      sessionId: session._id,
      openid: userId,
      role: 'user',
      content: initialMessage,
      type: 'text',
      createTime: new Date(),
      status: 'sent'
    };

    await db.collection('messages').add({ data: message });

    return { success: true, sessionId: session._id };
  } catch (error) {
    console.error('创建会话失败', error);
    return { success: false, error: error.message };
  }
};
```

### 3. 小程序前端实现

#### 聊天页面 (pages/message/service/index.js)

```javascript
Page({
  data: {
    sessionId: '',
    messages: [],
    inputValue: '',
    loading: false
  },
  onLoad(options) {
    const { sessionId } = options;
    this.setData({ sessionId });
    this.initMessages();
    this.listenMessages();
  },
  // 初始化消息列表
  async initMessages() {
    const { sessionId } = this.data;
    wx.showLoading({ title: '加载中...' });
    
    try {
      const db = wx.cloud.database();
      const res = await db.collection('messages')
        .where({ sessionId })
        .orderBy('createTime', 'asc')
        .get();
      
      this.setData({ messages: res.data });
      this.scrollToBottom();
    } catch (error) {
      console.error('加载消息失败', error);
      wx.showToast({ title: '加载消息失败', icon: 'none' });
    } finally {
      wx.hideLoading();
    }
  },
  // 监听消息变化
  listenMessages() {
    const { sessionId } = this.data;
    const db = wx.cloud.database();
    
    // 监听消息集合变化
    this.messageWatch = db.collection('messages')
      .where({ sessionId })
      .watch({
        onChange: (snapshot) => {
          if (snapshot.docChanges.length > 0) {
            // 处理新增消息
            const newMessages = snapshot.docChanges
              .filter(change => change.type === 'add')
              .map(change => change.doc);
            
            if (newMessages.length > 0) {
              const messages = [...this.data.messages, ...newMessages];
              this.setData({ messages });
              this.scrollToBottom();
            }
          }
        },
        onError: (error) => {
          console.error('监听消息失败', error);
        }
      });
  },
  // 发送消息
  async sendMessage() {
    const { sessionId, inputValue } = this.data;
    
    if (!inputValue.trim()) return;
    
    this.setData({ loading: true, inputValue: '' });
    
    try {
      const res = await wx.cloud.callFunction({
        name: 'sendMessage',
        data: {
          sessionId,
          content: inputValue.trim(),
          type: 'text'
        }
      });
      
      if (!res.result.success) {
        wx.showToast({ title: '发送失败', icon: 'none' });
      }
    } catch (error) {
      console.error('发送消息失败', error);
      wx.showToast({ title: '发送失败', icon: 'none' });
    } finally {
      this.setData({ loading: false });
    }
  },
  // 输入变化
  onInputChange(e) {
    this.setData({ inputValue: e.detail.value });
  },
  // 滚动到底部
  scrollToBottom() {
    setTimeout(() => {
      const query = wx.createSelectorQuery();
      query.select('#message-list').boundingClientRect();
      query.select('#message-list').scrollOffset();
      query.exec((res) => {
        if (res[0]) {
          wx.createSelectorQuery().select('#message-list').scrollIntoView({
            scrollTop: res[1].scrollTop + res[0].height,
            animated: true
          });
        }
      });
    }, 100);
  },
  // 页面卸载时取消监听
  onUnload() {
    if (this.messageWatch) {
      this.messageWatch.close();
    }
  }
});
```

### 4. 客服工作台实现

#### 客服登录页面

- 客服通过账号密码或微信登录
- 登录后更新客服状态为在线

#### 会话管理

- **我的会话**：显示分配给自己的会话列表
- **所有会话**：显示所有会话（可查看但有明确标识）
- **未分配会话**：显示待分配的新会话
- 会话列表按最后消息时间排序，未读消息优先

#### 消息回复

- 文本消息回复
- 图片、语音等多媒体消息回复
- 快捷回复模板

### 5. 客服分配机制

#### 分配策略

- **轮询分配**：新会话按顺序分配给在线客服
- **负载均衡**：优先分配给活跃会话数最少的客服
- **手动分配**：支持客服手动接管会话

#### 避免重复回复的机制

- 会话详情页面明确显示会话分配状态
- 客服只能回复分配给自己的会话
- 会话列表中明确标识会话归属
- 提供会话锁定机制，防止同时编辑

#### 客服状态管理

- 客服登录时更新状态为在线
- 客服登出时更新状态为离线
- 长时间无操作时自动设置为忙碌

### 6. 关键技术点

#### 1. 实时消息推送

- 使用云数据库的 **watch** 方法监听消息变化
- 当有新消息时，自动更新前端界面

#### 2. 消息状态管理

- 实现消息的发送、送达、已读状态
- 显示消息发送状态，提升用户体验

#### 3. 消息分页加载

- 实现消息的分页加载，避免一次性加载过多消息
- 滚动到底部时加载更多历史消息

#### 4. 客服分配机制

- 实现客服的自动分配
- 支持手动分配和转接功能

#### 5. 消息存储和清理

- 合理设置消息存储策略
- 定期清理过期消息，优化数据库性能

### 7. 优势与限制

#### 优势

- **实时性**：利用云数据库的实时推送能力，消息传递延迟低
- **易于部署**：基于云开发，无需搭建服务器
- **成本低**：云开发有免费额度，适合小型应用
- **安全可靠**：云开发提供了安全的认证和授权机制
- **客服分配合理**：避免两个客服重复回复同一客户

#### 限制

- **消息长度**：云数据库对文档大小有限制，消息内容不宜过长
- **并发限制**：云开发有并发请求限制，高并发场景可能需要优化
- **历史消息**：大量历史消息可能影响加载速度，需要分页处理

### 8. 优化建议

1. **消息缓存**：前端实现消息缓存，减少重复请求
2. **消息压缩**：对消息内容进行压缩，减少数据传输量
3. **离线消息**：实现离线消息功能，确保用户不会错过消息
4. **消息加密**：对敏感消息进行加密，保护用户隐私
5. **监控告警**：设置消息处理异常的监控和告警机制
6. **客服绩效统计**：统计客服的响应时间、解决率等指标，优化客服服务质量
