## 售后类型选择界面（2026年4月实测）

- 用户在订单详情页点击“申请售后”按钮后，弹窗展示售后类型选择。
- 主要类型有：
  1. 退款/退货（推荐，建议先与商家协商一致）
  2. 换货（需要更换商品）
- 用户需先选择售后类型，再进入下一步选择具体原因、填写凭证等。
- 该弹窗会展示本次售后申请的商品信息。

> 注：不同订单、类目、活动，售后类型可能略有差异，但主流流程均为“先选类型，再选原因”。

### 多商品订单的售后入口展示规则（2026年4月实测）

#### 订单列表页
- **不显示**“申请售后”按钮
- 显示“查看物流”、“加入购物车”等操作按钮
- 每个商品卡片明确显示“支持7天无理由”或“不支持7天无理由”标识

#### 订单详情页
- **每个商品卡片**独立显示“申请售后”按钮
- 每个商品卡片显示“加入购物车”按钮
- 每个商品卡片明确显示“支持7天无理由”或“不支持7天无理由”标识
- 用户点击单个商品的“申请售后”按钮，进入该商品的售后申请流程

> 注：多商品订单采用**按商品维度申请售后**的模式，而非整单申请，这样更灵活且符合实际售后场景。

## 退款/退货类型下的细分流程（2026年4月实测）

- 用户选择“退款/退货”后，会弹出“退款”和“退货退款”两个标签页：
  - 退款：不需要退货，仅退款。
  - 退货退款：需要退货并退款。

### 退款标签页

- 首先需选择“未收到货”或“已收到货”。
- 以“未收到货”为例：
  - 展示原因列表：
    - 拍错/多拍/不喜欢
    - 空包裹
    - 未按约定时间发货
    - 快递/物流一直未送到
    - 快递/物流无跟踪记录
    - 商品破损已拒签
  - 其中：
    - 选择“空包裹”“商品破损已拒签”时，下一步会先进入“上传凭证/说明”页面（强制要求上传图片/视频凭证和文字说明），再进入“确认退款信息”页面。
    - 选择“拍错/多拍/不喜欢”“未按约定时间发货”“快递/物流一直未送到”“快递/物流无跟踪记录”时，直接进入“确认退款信息”页面，不强制要求上传凭证和说明。

### 退款标签页 - 已收到货原因及凭证要求（2026年4月实测）

- 选择“退款”>“已收到货”后，原因列表包括：
  1. 协商一致退款
  2. 大小/尺寸与商品描述不符
  3. 颜色/图案/款式与商品描述不符
  4. 材质与商品描述不符
  5. 商品褪色/掉色/发黑等
  6. 质量问题（掉钻，掉胶，配件掉落等）
  7. 少件/漏发
  8. 包装/商品破损/污渍
  9. 未按约定时间发货
  10. 卖家发错货
- 不同原因对应的凭证/说明要求：
  - 直接进入“确认退款信息”页面（不强制上传凭证/说明）：
    - 协商一致退款
    - 未按约定时间发货
  - 需先进入“上传凭证/说明”页面（图片/视频凭证必填，说明必填）：
    - 大小/尺寸与商品描述不符
    - 颜色/图案/款式与商品描述不符
    - 材质与商品描述不符
    - 商品褪色/掉色/发黑等
    - 质量问题（掉钻，掉胶，配件掉落等）
    - 少件/漏发
    - 包装/商品破损/污渍
    - 卖家发错货
- 流程示例：
  1. 选择原因
  2. 若需上传凭证/说明，则进入上传页面，按要求填写
  3. 进入“确认退款信息”页面，填写退款金额，提交申请

> 注：不同原因的凭证要求和页面分支会动态变化，部分原因仅需说明，部分需强制图片/视频凭证。

### 退款申请流程页面顺序

1. 选择售后类型（退款/退货退款）
2. 选择货物状态（未收到货/已收到货）

4. （部分原因）上传凭证/说明（必填）
5. 确认退款信息，填写退款金额，提交申请

> 注：不同原因会动态决定是否强制要求上传凭证和说明，页面流程会有分支。

    - 协商一致退款
    - 不想要了
    - 未按约定时间发货
  - **需先进入“上传凭证/说明”页面（图片/视频凭证必填，说明必填）**：
    - 大小/尺寸与商品描述不符
    - 颜色/图案/款式与商品描述不符
    - 材质与商品描述不符

#### 上传凭证/说明页面（强制上传场景）

当选择需要强制上传凭证的原因时，会先进入“上传凭证/说明”页面，页面内容包括：

1. **图片/视频凭证（必填）**：
   - 提示“为了更快的帮助您处理问题，请上传说明/凭证”
   - 可上传最多9个文件
   - 标注为“必填”项
2. **详细说明（必填）**：
   - 文本输入框，最多200字
   - 提示“必填，请您填写文字说明”
   - 标注为“必填”项
3. **操作按钮**：
   - 上一步：返回上一页
   - 下一步：进入“确认退款信息”页面（需完成必填项才能点击）

#### 确认退款信息页面（强制上传后）

完成凭证/说明上传后，进入“确认退款信息”页面，与直接进入的页面相比，有以下特点：

1. **退款原因**：显示所选原因，并提示“若此原因属实，运费由卖家承担”
2. **上传描述和凭证**：显示已上传的凭证数量（标注为“\*”）
3. **退货方式**：自行寄回
4. **运费说明**：显示“运费卖家承担，售后结束后为您转账运费”

- 退货退款申请流程页面顺序：
  1. 选择售后类型（退款/退货退款）
  2. 选择“退货退款”标签
  3. 选择退货退款原因
  4. （部分原因）上传凭证/说明（按要求填写）
  5. 确认退款信息（含退款金额、退货方式、运费说明等）
  6. 提交申请

## 换货类型流程（2026年4月实测）

## 六、运费归属规则

### 1. 买家承担运费的情况

- 7天无理由退货退款、7天无理由换货（商品支持七天无理由退换货）
- 不想要了、拍错了等

### 2. 卖家承担运费的情况

- 商品质量问题
- 商家责任（发错货、少件漏发、包装破损）
- 物流异常等问题

## 七、售后时效规则

### 1. 七天无理由退货退款

- **适用条件**：商品支持七天无理由退换货 + 买家个人原因（不想要了、拍错了等）
- **起算时间**：签收时间
- **有效期限**：7天
- **凭证要求**：选填
- **运费归属**：买家承担

### 2. 质量问题售后

- **适用条件**：商品存在质量问题（褪色、掉钻、掉胶等）
- **起算时间**：签收时间
- **有效期限**：15天
- **凭证要求**：必填（图片+视频+说明）
- **运费归属**：卖家承担

### 3. 描述不符/商家责任售后

- **适用条件**：商品与描述不符、发错货、少件漏发、包装破损等
- **起算时间**：签收时间
- **有效期限**：7天
- **凭证要求**：必填（图片+视频+说明）
- **运费归属**：卖家承担

### 4. 物流异常售后

- **适用条件**：未收到货但物流异常（空包裹、物流一直未到等）
- **起算时间**：无需签收
- **有效期限**：发货后30天内
- **凭证要求**：部分必填（空包裹、商品破损已拒签必填）
- **运费归属**：卖家承担

### 5. 自动确认收货

- **时间**：发货后3天（可配置）
- **条件**：无物流更新时自动确认

## 八、售后申请规则

### 1. 售后申请入口条件

#### 1.1 订单状态限制
| 订单状态 | 是否可申请售后 | 说明 |
| :--- | :--- | :--- |
| 待付款 | ✗ | 订单未付款，无法申请售后 |
| 待发货 | ✗ | 商品未发货，无法申请售后 |
| 待收货 | ✓ | 可申请退款（未收到货场景） |
| 交易成功 | ✓ | 可申请所有类型售后 |
| 已取消 | ✗ | 订单已取消，无法申请售后 |

#### 1.2 售后时效限制
- 必须在对应的售后有效期内提交申请
- 超过时效后，"申请售后"按钮将不再显示
- 系统在售后申请页面会显示剩余售后时限

#### 1.3 售后申请数量限制
- 同一商品只能申请一次售后
- 同一订单可对多个商品分别申请售后
- 已完成或已取消的售后申请，不能再次申请

### 2. 退款金额规则

#### 2.1 退款金额计算
- **全额退款**：商品金额 + 运费（卖家责任时）
- **部分退款**：仅退还部分商品金额（需买家填写具体金额）

#### 2.2 退款金额限制
- **最低金额**：0.01元（不能为0）
- **最高金额**：商品实付金额 + 运费（如有）

#### 2.3 退款金额填写规则
- 退款申请：默认填写商品实付金额，可修改
- 退货退款申请：默认填写商品实付金额，可修改

### 3. 异常情况处理

#### 3.1 物流长时间不更新
- **触发条件**：物流信息超过7天未更新
- **处理方式**：系统自动提醒买家和商家，可申请客服介入

#### 3.2 买家不确认收货
- **触发条件**：商家寄回商品后，买家超过7天未确认收货
- **处理方式**：系统自动确认收货，售后流程结束

#### 3.3 商家超时未处理
- **审核超时**：超过48小时未审核，系统自动拒绝申请
- **寄回超时**：超过48小时未填写寄回物流，系统自动提醒商家

## 九、售后成功后的订单状态变化

### 1. 退款成功
- 订单状态保持"交易成功"
- 订单详情页显示"已退款"标识
- 退款金额原路返回给买家

### 2. 退货退款成功
- 订单状态保持"交易成功"
- 订单详情页显示"已退款"标识
- 退款金额原路返回给买家

### 3. 换货成功
- 订单状态保持"交易成功"
- 订单详情页显示"已换货"标识
- 记录换货商品信息和物流信息

## 十、售后服务评价（第三阶段）

### 1. 评价触发条件
- 售后流程完成后7天内
- 买家未评价时，系统会主动推送评价提醒

### 2. 评价内容
- **服务态度**：1-5星评分
- **处理速度**：1-5星评分
- **处理结果满意度**：1-5星评分
- **评价内容**：文字描述（选填，最多200字）

### 3. 评价展示
- 评价内容展示在店铺页面
- 评价内容展示在商品详情页
- 商家可在后台查看所有评价

## 九、售后状态流转规则

### 1. 状态定义

| 状态值 | 状态名称 | 说明 |
| :--- | :--- | :--- |
| `pending` | 待审核 | 买家提交售后申请后，等待商家审核 |
| `processing` | 处理中 | 商家已同意申请，进入处理流程 |
| `buyer_shipping` | 买家退货中 | 退货退款/换货场景，买家已填写退货物流信息 |
| `seller_reviewing` | 商家验货中 | 商家收到退货，正在验货 |
| `seller_returning` | 商家寄回中 | 验货不通过，商家将商品寄回给买家 |
| `completed` | 已完成 | 售后流程结束（退款成功/换货完成/商品已寄回） |
| `cancelled` | 已取消 | 买家主动取消或商家拒绝申请（不含验货不通过场景） |

### 2. 状态流转图

#### 退款/退货退款流程
```
买家提交申请
    ↓
pending（待审核）
    ↓ 商家拒绝
cancelled（已取消）
    ↑
pending（待审核）
    ↓ 商家同意
processing（处理中）
    ↓ 退款申请
completed（已完成）
    ↑
processing（处理中）
    ↓ 退货退款申请
buyer_shipping（买家退货中）
    ↓ 买家填写物流
seller_reviewing（商家验货中）
    ↓ 验货通过
completed（已完成）
    ↑
seller_reviewing（商家验货中）
    ↓ 验货不通过
seller_returning（商家寄回中）
    ↓ 商家填写寄回物流
seller_returning（商家寄回中）
    ↓ 买家确认收货
completed（已完成）
```

#### 换货流程
```
买家提交换货申请
    ↓
pending（待审核）
    ↓ 商家拒绝
cancelled（已取消）
    ↑
pending（待审核）
    ↓ 商家同意
processing（处理中）
    ↓ 买家填写退货物流
buyer_shipping（买家退货中）
    ↓ 商家确认收到退货
seller_reviewing（商家验货中）
    ↓ 验货通过，商家发新货
completed（已完成）
    ↑
seller_reviewing（商家验货中）
    ↓ 验货不通过
seller_returning（商家寄回中）
    ↓ 商家填写寄回物流
seller_returning（商家寄回中）
    ↓ 买家确认收货
completed（已完成）
```

### 3. 状态流转触发条件

| 当前状态 | 触发动作 | 下一状态 |
| :--- | :--- | :--- |
| - | 买家提交售后申请（退款/退货退款/换货） | pending |
| pending | 商家同意申请 | processing |
| pending | 商家拒绝申请 | cancelled |
| processing | 退款申请，退款完成 | completed |
| processing | 退货退款，买家填写物流 | buyer_shipping |
| processing | 换货申请，买家填写退货物流 | buyer_shipping |
| buyer_shipping | 商家确认收到退货 | seller_reviewing |
| seller_reviewing | 退货退款，验货通过，退款完成 | completed |
| seller_reviewing | 换货，验货通过，商家发新货 | completed |
| seller_reviewing | 验货不通过 | seller_returning |
| seller_returning | 商家填写寄回物流 | seller_returning |
| seller_returning | 买家确认收到商品 | completed |
| pending/processing | 买家主动取消 | cancelled |

### 4. 商家寄回流程（验货不通过时）

> 说明：为简化流程，验货不通过时商家直接将商品寄回给买家，无需申诉环节。这样可以避免买家不操作导致流程卡住，提升用户体验。

#### 4.1 触发条件
- 商家验货不通过

#### 4.2 寄回流程步骤

1. **商家验货不通过通知**：
   - 商家验货不通过时，需上传验货凭证（图片/视频）
   - 填写拒绝理由（如：商品已使用、影响二次销售等）
   - 系统自动通知买家验货结果和拒绝理由
   - 状态变更为 seller_returning

2. **商家寄回商品**：
   - 商家需在48小时内将商品寄回给买家
   - 填写寄回物流单号
   - 系统通知买家商家已寄回

3. **买家确认收货**：
   - 买家收到商品后确认收货
   - 状态变更为 completed
   - 售后流程结束

#### 4.3 寄回流程责任说明

| 责任方 | 责任内容 |
| :--- | :--- |
| 商家 | 负责将商品寄回给买家，承担寄回运费 |
| 买家 | 负责签收商品 |

> 注：商家寄回运费由商家承担，因为买家已经支付了商品费用，验货不通过是商家的判断，不应让买家承担额外费用。

#### 4.4 页面设计

**商家端寄回页面**：
- 显示需要寄回的商品信息
- 显示之前上传的验货凭证和拒绝理由
- 物流单号输入框（必填）
- 确认寄回按钮
- 系统自动填充买家收货地址

**买家端寄回物流页面**：
- 显示商家验货不通过的理由和凭证
- 显示商家寄回的物流信息
- 物流轨迹展示
- 确认收货按钮

#### 4.5 寄回时限
- 商家寄回时限：验货不通过后48小时内

## 十、商家端操作流程

### 1. 商家售后管理页面

#### 页面结构

1. **筛选区域**：
   - 按状态筛选：全部、待审核、处理中、买家退货中、商家验货中、商家寄回中、已完成、已取消
   - 按售后类型筛选：全部、退款、退货退款、换货
   - 按时间筛选：今天、近7天、近30天
   - 搜索框：支持订单编号、买家昵称搜索

2. **售后列表**：
   - 订单编号（可点击查看订单详情）
   - 买家信息（头像、昵称）
   - 商品信息（图片、名称、规格）
   - 售后类型、售后原因
   - 申请金额、申请时间
   - 当前状态

3. **操作按钮**：
   - 待审核状态：同意、拒绝
   - 处理中状态：查看详情、填写处理备注
   - 买家退货中：确认收货
   - 商家验货中：确认通过、拒绝
   - 商家寄回中：填写寄回物流

### 2. 商家审核操作

#### 同意申请

1. 点击"同意"按钮
2. 弹出确认弹窗：
   - 显示售后申请详情
   - 确认退款金额
   - 可填写处理备注（选填）
3. 点击"确认同意"：
   - 状态变更为 processing
   - 系统自动发送通知给买家
   - 退款申请：直接进入退款流程
   - 退货退款申请：等待买家填写退货物流

#### 拒绝申请

1. 点击"拒绝"按钮
2. 弹出拒绝原因选择弹窗：
   - 拒绝原因列表：
     - 商品已影响二次销售
     - 凭证不足，无法核实
     - 超出售后时效
     - 其他原因（需填写说明）
   - 需填写拒绝说明（必填）
3. 点击"确认拒绝"：
   - 状态变更为 cancelled
   - 系统自动发送通知给买家，包含拒绝原因

### 3. 退货退款处理流程

#### 买家退货中

1. 商家可查看买家填写的退货物流信息
2. 系统自动追踪物流状态

#### 商家验货

1. 商家收到退货后点击"确认收货"
2. 状态变更为 seller_reviewing
3. 验货通过：
   - 点击"确认通过"
   - 执行退款操作
   - 状态变更为 completed
4. 验货不通过：
   - 点击"拒绝"
   - 填写拒绝原因（必填）
   - 状态变更为 seller_returning

### 4. 换货处理流程

1. 商家同意换货申请后，状态变更为 processing
2. 商家确认收到买家退货后，状态变更为 seller_reviewing
3. 验货通过：
   - 点击"确认通过"
   - 安排换货商品发货
   - 填写换货物流信息
   - 状态变更为 completed
   - 系统通知买家
4. 验货不通过：
   - 点击"拒绝"
   - 填写拒绝原因（必填）
   - 状态变更为 seller_returning

### 5. 商家端通知机制

| 触发事件 | 通知方式 | 通知内容 |
| :--- | :--- | :--- |
| 买家提交售后申请 | 消息推送+短信 | 有新的售后申请待处理 |
| 买家填写退货物流 | 消息推送 | 买家已填写退货物流 |
| 售后状态变更 | 消息推送 | 售后申请状态已更新 |

## 十一、买家端售后进度查看

### 1. 售后详情页面

1. **售后信息**：
   - 售后类型、售后原因
   - 申请金额、申请时间
   - 当前状态及状态变更时间

2. **物流信息**（退货退款场景）：
   - 退货物流单号
   - 物流轨迹展示

3. **操作按钮**：
   - 待审核状态：取消申请
   - 买家退货中：修改物流信息
   - 已完成/已取消：查看详情

### 2. 买家取消申请

#### 2.1 可取消状态
| 状态值 | 状态名称 | 是否可取消 | 说明 |
| :--- | :--- | :--- | :--- |
| `pending` | 待审核 | ✓ | 商家尚未处理，买家可随时取消 |
| `processing` | 处理中 | ✓ | 商家已同意，但买家尚未填写退货物流 |
| `buyer_shipping` | 买家退货中 | ✓ | 买家已填写物流，但商家尚未确认收货 |
| `seller_reviewing` | 商家验货中 | ✗ | 商家已收到退货，正在验货 |
| `seller_returning` | 商家寄回中 | ✗ | 商家正在将商品寄回给买家 |
| `completed` | 已完成 | ✗ | 售后流程已结束 |
| `cancelled` | 已取消 | ✗ | 已取消的申请不可重复取消 |

#### 2.2 取消操作流程
- 点击"取消申请"按钮
- 弹出确认弹窗，提示"确定要取消售后申请吗？"
- 确认后状态变更为 cancelled
- 系统通知商家买家已取消申请

## 十五、注意事项

1. **历史订单**：可手动删除，不用管
2. **商品维度配置**：不同商品可以有不同的七天无理由退换货策略
3. **凭证安全**：凭证图片/视频需要加水印，防止滥用
4. **时限控制**：严格按照售后时效规则校验
5. **运费说明**：在页面清晰展示运费归属规则
