# 订单状态与标签对应关系

## 快递运输 (express)

- **待支付**：status = "pending"
- **待发货**：status = "paid"
- **待收货**：status = "shipping"

  待收货的订单卡片，在物流信息为运输中，会显示已发货；在物流信息为已签收，会显示已签收，待确认收货。
- **已完成**：status = "completed"
- **售后**：status = "refund" 或 "refund\_completed"
- **已取消**：status = "cancelled"

## 上门自提 (pickup)

- **待支付**：status = "pending"
- **待自提**：status = "paid"
- **已完成**：status = "completed"
- **已取消**：status = "cancelled"

## 同城配送 (local)

- **待支付**：status = "pending"
- **待配送**：status = "paid"
- **配送中**：status = "shipping"
- **已完成**：status = "completed"
- **已取消**：status = "cancelled"

## 状态映射关系

- **paid** 状态：
  - 快递运输 → 待发货
  - 上门自提 → 待自提
  - 同城配送 → 待配送
- **shipping** 状态：
  - 快递运输 → 待收货
  - 同城配送 → 配送中

## 订单生命周期

1. **创建订单**：status = "pending"（待支付）
2. **支付成功**：status = "paid"（根据配送方式显示为待发货/待自提/待配送）
3. **商家发货**：status = "shipping"（根据配送方式显示为待收货/配送中）
4. **确认收货**：status = "completed"（已完成）
5. **申请售后**：status = "refund"（售后中）
6. **售后完成**：status = "refund\_completed"（售后完成）
7. **取消订单**：status = "cancelled"（已取消）
8. **订单过期**：status = "cancelled"（已取消）

## 备注

- 待发货标签仅在快递运输方式下显示，对应status为paid的订单
- 待收货标签对应status为shipping的订单
- 待自提标签仅在上门自提方式下显示，对应status为paid的订单
- 待配送标签仅在同城配送方式下显示，对应status为paid的订单
- 配送中标签仅在同城配送方式下显示，对应status为shipping的订单
