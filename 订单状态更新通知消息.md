# 订单状态更新通知消息

## 通知类型
- **小程序通知**：通过小程序内的消息通知功能
- **微信模板消息**：通过微信公众号发送的模板消息（重要消息）

## 订单状态流转通知

### 1. 下单成功（创建订单）
- **通知对象**：用户
- **小程序通知**：
  - 标题：订单创建成功
  - 内容：您的订单 #订单号 已创建成功，请在30分钟内完成支付
- **微信模板消息**：否（一般重要）
- **触发条件**：订单创建成功，状态变为 pending

### 2. 支付成功
- **通知对象**：用户、管理员
- **小程序通知**：
  - 用户：您的订单 #订单号 支付成功，我们正在处理
  - 管理员：新订单 #订单号 已支付，请注意处理
- **微信模板消息**：
  - 管理员：新订单通知（包含订单详情、支付金额、联系方式）
- **触发条件**：用户支付成功，状态变为 paid

### 3. 发货/开始配送
- **通知对象**：用户
- **小程序通知**：
  - 快递运输：您的订单 #订单号 已发货，物流单号：物流单号
  - 同城配送：您的订单 #订单号 已开始配送，请保持手机畅通
- **微信模板消息**：
  - 快递运输：发货通知（包含物流单号、预计送达时间）
- **触发条件**：商家发货，状态变为 shipping

### 4. 确认收货/自提完成
- **通知对象**：用户、管理员
- **小程序通知**：
  - 用户：您的订单 #订单号 已完成，感谢您的购买
  - 管理员：订单 #订单号 已完成，交易成功
- **微信模板消息**：否（一般重要）
- **触发条件**：用户确认收货或自提完成，状态变为 completed

### 5. 申请售后
- **通知对象**：用户、管理员
- **小程序通知**：
  - 用户：您的售后申请已提交，我们会尽快处理
  - 管理员：订单 #订单号 有新的售后申请，请注意处理
- **微信模板消息**：
  - 管理员：售后申请通知（包含售后原因、订单详情）
- **触发条件**：用户申请售后，状态变为 refund

### 6. 售后完成
- **通知对象**：用户
- **小程序通知**：您的售后申请已处理完成，如有疑问请联系客服
- **微信模板消息**：
  - 退款成功：退款通知（包含退款金额、到账时间）
- **触发条件**：售后处理完成，状态变为 refund_completed

### 7. 主动取消订单
- **通知对象**：用户、管理员
- **小程序通知**：
  - 用户：您的订单 #订单号 已取消
  - 管理员：订单 #订单号 已被取消
- **微信模板消息**：否（一般重要）
- **触发条件**：用户或管理员主动取消订单，状态变为 cancelled

### 8. 订单过期
- **通知对象**：用户
- **小程序通知**：您的订单 #订单号 因超时未支付已自动取消
- **微信模板消息**：否（一般重要）
- **触发条件**：订单超过支付时间，状态变为 cancelled

## 通知实现建议

### 小程序通知
- 使用 `wx.cloud.callFunction` 调用云函数发送通知
- 利用微信小程序的订阅消息功能
- 通知内容应简洁明了，包含订单号和关键信息

### 微信模板消息
- 需要用户预先授权
- 模板消息应包含详细的订单信息
- 仅用于重要通知，如新订单、发货、退款等

### 通知触发机制
- 在 `updateOrderStatus` 云函数中集成通知逻辑
- 根据操作类型和状态变化自动触发相应的通知
- 确保通知的及时性和准确性

## 通知内容模板

### 新订单通知（管理员）
```
【新订单通知】
订单号：{orderNumber}
支付金额：{amount}元
配送方式：{deliveryType}
下单时间：{createTime}
联系人：{name}
联系电话：{phone}
请及时处理订单！
```

### 发货通知（用户）
```
【发货通知】
您的订单 #订单号 已发货
物流单号：{trackingNumber}
预计送达时间：{estimatedTime}
点击查看物流详情
```

### 退款成功通知（用户）
```
【退款通知】
您的订单 #订单号 退款成功
退款金额：{amount}元
预计到账时间：1-3个工作日
如有疑问请联系客服
```
