# 售后业务规则表（V2）

## 1. 规则目标
- 支持 7 天无理由能力按类型独立开关：退货退款、换货、仅退款。
- 支持按订单明细行（商品行）发起和处理售后。

## 2. 售后类型定义
| 类型编码 | 展示名称 | 是否无理由 | 是否需要退货 | 是否允许直接退款 | 备注 |
|---|---|---|---|---|---|
| refund | 退货退款（无理由） | 是 | 是 | 否 | 7 天内可申请 |
| no_reason_exchange | 换货（无理由） | 是 | 是 | 否 | 7 天内可申请 |
| exchange | 换货（质量问题） | 否 | 是 | 否 | 质量问题售后类型 |
| quality_refund | 质量问题退款 | 否 | 可配置 | 是 | 是否退货可按品类策略决定 |
| only_refund | 仅退款（无理由） | 是（受开关控制） | 否 | 是 | 7 天内可申请；默认关闭 |

## 3. 开关与时效配置（settings.afterSalesTimeConfig）
```json
{
  "supportNoReasonReturn": true,
  "supportNoReasonReturnRefund": true,
  "supportNoReasonExchange": true,
  "supportNoReasonOnlyRefund": false,
  "requireImageWhenNoReasonDisabled": false,
  "requireVideoWhenNoReasonDisabled": false,
  "requireImageWhenOnlyRefundNoReasonDisabled": false,
  "requireVideoWhenOnlyRefundNoReasonDisabled": false,
  "noReasonReturnDays": 7,
  "normalAfterSalesDays": 7,
  "qualityAfterSalesDays": 15,
  "autoConfirmReceiptDays": 3
}
```

说明：
- supportNoReasonReturn: 无理由总开关；为 false 时，三个无理由子项全部不可用。
- supportNoReasonReturnRefund: 控制是否开放“无理由退货退款”。
- supportNoReasonExchange: 控制是否开放“无理由换货（no_reason_exchange）”。
- supportNoReasonOnlyRefund: 控制是否开放“无理由仅退款（only_refund）”，默认 false。
- requireImageWhenNoReasonDisabled: 当 supportNoReasonReturn=false 时，是否必须上传图片。
- requireVideoWhenNoReasonDisabled: 当 supportNoReasonReturn=false 时，是否必须上传开箱视频。
- requireImageWhenOnlyRefundNoReasonDisabled: 当 supportNoReasonOnlyRefund=false 且申请 only_refund 时，是否必须上传图片。
- requireVideoWhenOnlyRefundNoReasonDisabled: 当 supportNoReasonOnlyRefund=false 且申请 only_refund 时，是否必须上传开箱视频。

## 4. 用户端可选项规则
### 4.1 当 supportNoReasonReturn = true
- 展示：
  - 退货退款（无理由）: 当 supportNoReasonReturnRefund = true
  - 换货（无理由）: 当 supportNoReasonExchange = true
  - 仅退款（无理由）: 当 supportNoReasonOnlyRefund = true
  - 质量问题退款
  - 换货（质量问题）
- 约束：三个无理由子开关不可同时关闭（总开关开启时至少保留一个可用类型）

### 4.2 当 supportNoReasonReturn = false
- 展示：
  - 退货退款（需提供理由）
  - 仅退款（需提供理由）
  - 质量问题退款
  - 换货（质量问题）
- 不展示：
  - 换货（无理由）
  - 退货退款（无理由）/仅退款（无理由）标签文案

### 4.3 仅退款（only_refund）
- 当 supportNoReasonReturn = true 且 supportNoReasonOnlyRefund = true 时，用户端可直接展示“仅退款（无理由）”。
- 当 supportNoReasonOnlyRefund = false 时，only_refund 仍可按“需提供理由”方式申请（不属于无理由场景）。
- 当 supportNoReasonOnlyRefund = false 且配置了凭证强制开关时，only_refund 必须按配置上传图片/开箱视频。

### 4.4 凭证强制规则
- 当 supportNoReasonReturn = false：
  - requireImageWhenNoReasonDisabled = true 时，申请必须上传图片。
  - requireVideoWhenNoReasonDisabled = true 时，申请必须上传开箱视频。
- 当 supportNoReasonOnlyRefund = false 且申请 only_refund：
  - requireImageWhenOnlyRefundNoReasonDisabled = true 时，申请必须上传图片。
  - requireVideoWhenOnlyRefundNoReasonDisabled = true 时，申请必须上传开箱视频。

## 5. 时效校验规则
- 售后时间基准：
  - 优先使用 logisticsState.checkTime（签收时间）。
  - 缺失时回退 receiptTime。
- 各类型时效：
  - no_reason_exchange：noReasonReturnDays。
  - refund / only_refund：根据是否属于无理由场景动态判定：
    - 无理由场景：noReasonReturnDays
    - 有理由场景：normalAfterSalesDays
  - 常规售后：normalAfterSalesDays。
  - 质量问题：qualityAfterSalesDays。

## 6. 明细行数量校验规则
- 每个 orderItemId 的累计售后数量不得超过购买数量。
- 同一 orderItemId 的处理中数量应锁定，避免并发超量申请。
- 校验时机：
  - 用户提交申请时。
  - 管理员审核通过前二次校验。

## 7. 管理员处理规则（按明细行）
- 行级动作：
  - approve（同意）
  - reject（拒绝）
  - complete（完成）
- 主单状态聚合：
  - 明细全部 completed/rejected/cancelled -> 主单完成或关闭。
  - 存在 approved/processing -> 主单 processing。
  - 明细全 rejected -> 主单 rejected。
  - 明细全 cancelled -> 主单 cancelled。

## 8. 风险控制（必须）
- 无理由仅退款默认关闭（supportNoReasonOnlyRefund = false）。
- 退款金额上限按明细行可退金额校验。
- 关键动作落 after_sales_logs 审计。
- 退货物流记录入 reverse_logistics。

## 9. 文案建议
- 无理由场景：
  - 按开关动态显示“7天无理由退货退款、7天无理由换货、7天无理由仅退款”。
- 质量问题场景：
  - 显示“质量问题退款、换货（质量问题）”。
- 类型文案与编码映射建议：
  - refund -> 退货退款（无理由）
  - no_reason_exchange -> 换货（无理由）
  - only_refund -> 仅退款（无理由）
  - exchange -> 换货（质量问题）
  - quality_refund -> 质量问题退款

## 10. 执行优先级
1. 后端校验优先，前端仅做引导。
2. 历史订单按 policySnapshot 规则处理，避免规则变更争议。
3. 审核口径以明细行为准，不按整单一刀切。
