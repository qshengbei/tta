# 云函数设计

## 1. 云函数列表

| 云函数名称 | 描述 | 功能 |
|-----------|------|------|
| updateOrderStatus | 更新订单状态 | 处理订单状态流转，支持支付、发货、确认收货、取消订单等操作 |
| sendNotification | 发送通知 | 发送小程序通知和微信模板消息，支持多种通知类型 |
| updateSettings | 更新系统设置 | 更新系统设置，如购买须知、自提地址等 |
| checkExpiredOrders | 检查过期订单 | 定期检查并处理超时未支付的订单 |
| getProductInfo | 获取商品信息 | 获取商品详细信息，包括库存、价格等 |
| createOrder | 创建订单 | 创建新订单，处理库存扣减等逻辑 |
| processPayment | 处理支付 | 处理支付回调，更新订单状态 |
| uploadImage | 上传图片 | 处理图片上传，生成云存储链接 |

## 2. updateOrderStatus 云函数

### 2.1 功能描述

更新订单状态，支持完整的订单生命周期状态流转，包括支付、发货、确认收货、取消订单、申请售后、处理售后等操作。

### 2.2 参数说明

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| orderId | string | 是 | 订单ID |
| operation | string | 是 | 操作类型：pay, ship, confirm, cancel, applyAfterSales, processAfterSales |
| params | object | 否 | 附加参数，根据操作类型不同而不同 |

### 2.3 操作类型说明

| 操作类型 | 描述 | 参数说明 |
|----------|------|----------|
| pay | 支付订单 | 无特殊参数 |
| ship | 发货 | trackingNumber: 物流单号（快递配送时必需） |
| confirm | 确认收货 | 无特殊参数 |
| cancel | 取消订单 | cancelReason: 取消原因 |
| applyAfterSales | 申请售后 | reason: 售后原因 |
| processAfterSales | 处理售后 | result: 处理结果 |

### 2.4 返回结果

| 字段名 | 类型 | 描述 |
|--------|------|------|
| success | boolean | 操作是否成功 |
| message | string | 操作结果消息 |
| data | object | 操作结果数据 |
| error | string | 错误信息（失败时返回） |

### 2.5 状态流转

| 当前状态 | 操作 | 新状态 | 适用配送方式 |
|----------|------|--------|--------------|
| pending | pay | paid | 所有 |
| paid | ship | shipping | 快递、同城配送 |
| paid | confirm | completed | 上门自提 |
| shipping | confirm | completed | 快递、同城配送 |
| any | cancel | cancelled | 所有（已完成和已取消订单除外） |
| completed | applyAfterSales | completed (afterSalesStatus: pending) | 所有 |
| completed (afterSalesStatus: pending/processing) | processAfterSales | completed (afterSalesStatus: completed) | 所有 |

## 3. sendNotification 云函数

### 3.1 功能描述

发送通知消息，支持小程序通知和微信模板消息，根据通知类型和场景选择对应的模板。

### 3.2 参数说明

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| notificationType | string | 是 | 通知类型：orderStatusChange, activity, system, restock |
| targetUsers | array | 是 | 目标用户openid数组 |
| templateId | string | 否 | 模板消息ID（可选） |
| data | object | 是 | 通知数据，包含模板变量 |
| extras | object | 否 | 额外参数 |

### 3.3 通知类型说明

| 通知类型 | 描述 | 场景示例 |
|----------|------|----------|
| orderStatusChange | 订单状态变化通知 | 支付成功、发货、确认收货等 |
| activity | 活动通知 | 新品上架、促销活动等 |
| system | 系统通知 | 系统更新、维护通知等 |
| restock | 补货通知 | 商品补货、到货提醒等 |

### 3.4 返回结果

| 字段名 | 类型 | 描述 |
|--------|------|------|
| success | boolean | 操作是否成功 |
| message | string | 操作结果消息 |
| data | array | 每个用户的通知发送结果 |
| error | string | 错误信息（失败时返回） |

| settings | object | 是 | 要更新的设置字段 |

### 4.3 支持的设置字段

| 字段名 | 类型 | 描述 |
|--------|------|------|
| pickupLocation | string | 自提地址 |
| beginTime | string | 可自提与配送开始时间 |
| endTime | string | 可自提与配送结束时间 |

| deliveryRules | array | 同城配送配送费配置 |

### 4.4 返回结果

| 字段名 | 类型 | 描述 |
|--------|------|------|
| success | boolean | 操作是否成功 |
| message | string | 操作结果消息 |
| data | object | 更新后的设置数据 |
| error | string | 错误信息（失败时返回） |

## 10. 云函数调用示例

### 10.1 调用 updateOrderStatus 云函数

```javascript
wx.cloud.callFunction({
  name: 'updateOrderStatus',
  data: {
    orderId: 'order123',
    operation: 'pay',
    params: {}
  },
  success: res => {
    console.log('订单支付成功:', res.result);
  },
  fail: err => {
    console.error('调用失败:', err);
  }
});
```

### 10.2 调用 sendNotification 云函数

```javascript
wx.cloud.callFunction({
  name: 'sendNotification',
  data: {
    notificationType: 'orderStatusChange',
    targetUsers: ['openid123'],
    data: {
      status: 'paid',
      orderNumber: '202604090001',
      amount: 99.99
    },
    extras: {
      orderId: 'order123'
    }
  },
  success: res => {
    console.log('通知发送成功:', res.result);
  },
  fail: err => {
    console.error('调用失败:', err);
  }
});
```

### 10.3 调用 updateSettings 云函数

```javascript
wx.cloud.callFunction({
```
