# 云开发 quickstart

这是云开发的快速启动指引，其中演示了如何上手使用云开发的三大基础能力：

- 数据库：一个既可在小程序前端操作，也能在云函数中读写的 JSON 文档型数据库
- 文件存储：在小程序前端直接上传/下载云端文件，在云开发控制台可视化管理
- 云函数：在云端运行的代码，微信私有协议天然鉴权，开发者只需编写业务逻辑代码

**问题**：腾讯位置服务地址解析接口签名验证失败
**原因**：在使用腾讯地图SDK时，`sig`参数的传递方式不正确

**修复详情**：
1. 初始化SDK时只传入`key`参数

**验证结果**：地址解析功能正常工作，能够成功获取自提地址的坐标


**问题**：地图圆形覆盖物描边颜色显示为黑色，无法正确显示设置的颜色
**原因**：使用了错误的属性名 `strokeColor` 来设置描边颜色，而微信小程序官方文档中圆形覆盖物的描边颜色属性名应该是 `color`
**解决方案**：将 `strokeColor` 改为 `color`，使用正确的属性名来设置描边颜色
**修复文件**：`miniprogram/pages/order-confirm/index.js`
**修复详情**：
1. 在生成圆形覆盖物时，将 `strokeColor` 属性改为 `color`
2. 使用标准的十六进制颜色格式设置描边颜色：淡绿色 `#00ff99`、淡黄色 `#ffff99`、淡红色 `#ff9999`
3. 保持填充颜色的设置，使用带透明度的十六进制颜色格式

**验证结果**：地图圆形覆盖物的描边颜色现在能够正确显示为设置的颜色，不再显示为黑色


**问题**：同城配送超过范围时仍然提示"当天已无配送时间"，配送距离为0.0km时配送费显示10元，商品金额和运费合计显示错误，地图缩放比例没重置

4. 时间初始化逻辑在距离计算之前执行，导致超过配送范围时仍然提示时间
**解决方案**：
1. 修改配送费计算的遍历顺序，从从小到大遍历，确保0.0km能匹配到最小的配送规则
2. 在totalPrice计算时使用Number()函数确保所有值都是数字类型，避免字符串拼接
3. 确保所有mapContext.moveToLocation调用都设置了scale: 15
4. 修改confirmDeliveryType函数的执行顺序，先计算距离和配送范围，然后再根据结果决定是否初始化自提时间

**问题**：购物车页面筛选面板间隔不一致，一级分类按钮变成长条状

3. 统一了筛选面板中各个元素之间的间隔
**修复文件**：
- `miniprogram/pages/cart/index.wxss`
- `miniprogram/pages/cart/index.wxml`

2. 为`.cart__filter-category-group`添加了Flexbox布局，使用`flex-direction: column`和`gap: 12rpx`
3. 为一级分类按钮添加了`cart__filter-option-level1`类，并设置了`align-self: flex-start`

**验证结果**：
